# 知识管理目录规范（03-领域）

> [!summary]
> `03-领域/` 以 Wiki 方式管理知识领域的学习笔记。目录即主题，folder-note 即 MOC。层级不超过 5 层，主题门槛 3+ 篇笔记。

## 层级上限

```
03-领域/                          ← 第 1 层
  {领域}/                         ← 第 2 层（如 31-人工智能学科）
    {主题}/                       ← 第 3 层（如 智能体工具）
      {子主题}/                   ← 第 4 层
        {笔记}.md                 ← 第 5 层（最深）
```

超过 5 层的路径必须合并中间层级。

## 目录 = 主题

一个目录就是一个独立主题。建目录的门槛：

- **当前 3+ 篇笔记**，或有明确的增长预期
- 不满门槛的，以单文件笔记放在父级 MOC 下，待扩展后再升级为目录

反面例子：

- `10-建筑学/` 只有 1 个文件 → 不应建子目录
- `chatgpt/` 只有 1 个文件 → 取消目录，文件上浮到父级

## 命名规则

| 层级 | 规则 | 示例 |
|------|------|------|
| 第 1 层（领域） | 保留数字前缀，维持排序 | `00-认知科学`、`31-人工智能学科` |
| 第 2 层及以下 | **主题名**，无数字前缀 | `认知术语`、`智能体工具`、`claude` |
| 中英文混用 | 中文为主，英文术语括号补充 | `智能体(Agent)`、`大语言模型(LLMs)` |
| 字母后缀 | 不使用 | 废弃 `(G)(P)(S)(K)(C)` 等标记 |

## 每个目录一个同名 MOC

```
31-人工智能学科/
  31-人工智能学科.md    ← folder-note = MOC
  AI术语/
    AI术语.md           ← folder-note = MOC
```

MOC 内容遵循 [[MOC-Wiki目录页规范]]：

- 仅保留目录树 + Wiki 链接 + 极少量说明
- 每个子主题附带一句话说明
- 不放正文、长表格、配置细节、操作步骤

### MOC 模板

```markdown
---
aliases: [检索别名]
tags: [领域标签]
description: 一句话
type: MOC
---

## 子主题
- [[子目录名]] — 一句话说明

## 单篇笔记
- [[笔记名]] — 一句话说明
```

## 收件箱

未归类的临时笔记统一放入 `03-领域/--草稿/`。定期整理，达到 3+ 篇就建主题目录。

## 变更规则（从旧结构迁移）

| 旧 | 新 | 原因 |
|----|-----|------|
| 二级目录带数字前缀（`00-认知术语`） | 去数字前缀（`认知术语`） | 领域已提供排序上下文 |
| 字母后缀（`00-艺术术语(G)`） | 去掉后缀（`艺术术语`） | 无检索价值 |
| 英文前缀（`it-知识体系`、`ai-大语言模型`） | 中文命名（`知识体系`、`大语言模型`） | 中文为主 |
| 裸 `--` 草稿目录 | 合并到 `--草稿/` | 收件箱唯一 |
| 过深层级（5 层+） | 合并中间层 | 层级上限 |

## 判断标准

一个 `03-领域` 下的目录结构是否合规：

1. 任意文件路径不超过 5 层（从 `03-领域/` 起算）。
2. 每个目录下至少有 3 个条目（子目录 + 单篇笔记），或已确认即将扩展。
3. 第 2 层及以下目录名不含数字前缀。
4. 每个目录存在同名 `.md` 作为 MOC。
5. MOC 是导航页，不是文章。
6. `--草稿/` 是唯一的收件箱，不存在其他 `--` 开头的临时目录。

## 压缩记忆

> 目录即主题，folder-note 即 MOC。5 层封顶，3 篇起建。领域层留数字，主题层去前缀。`--草稿` 统一收件箱。MOC 是地图不是文章。
