# 术语审计规范

> [!summary]
> 本规范用于统一 Vault 内术语目录、术语卡、Base 数据库、MOC 入口和分库边界的审计口径。整改项目可引用本规范，但不得在项目内另立长期规则源。

## 适用范围

本规范适用于 Vault 内承担“术语管理”职责的目录、入口页、术语卡、Base 数据库视图和相关链接关系。

| 对象 | 纳入审计 | 说明 |
|------|----------|------|
| 通用术语库 | 是 | `03-领域/00-概念术语/` 是通用术语主库 |
| 学科术语库 | 是 | AI、管理、哲学、艺术、认知、信息学科等领域下的术语目录 |
| 临时项目术语表 | 是 | 只审其是否应回流到领域术语库，不直接要求完整治理 |
| Base 数据库 | 是 | 只审术语相关 Base 的路径、筛选条件、字段口径 |
| 资源库/案例库 | 否 | 若放在术语目录内，按“目录边界错位”处理 |
| 普通知识笔记 | 条件纳入 | 仅当其 Frontmatter 或路径声明为术语时纳入 |

## 术语卡判定

术语卡是用于解释一个概念、缩写、方法、模型、对象或专业表达的稳定知识笔记。合格术语卡至少能回答：它是什么、边界是什么、和相近概念有什么不同、如何使用或识别。

| 类型 | 判定 |
|------|------|
| 概念定义 | 纳入术语卡 |
| 缩写解释 | 纳入术语卡 |
| 方法/模型说明 | 纳入术语卡 |
| 工具使用教程 | 不纳入，除非正文定位为术语解释 |
| 课程/项目临时词表 | 暂纳入候选，完成后应回流或归档 |
| 空白占位页 | 不视为合格术语卡，按质量问题处理 |

## 目录边界

`03-领域/00-概念术语/` 只承担通用术语库职责，不承担泛数据库看板、案例库、资源库、项目资料库职责。

| 内容 | 正确归属 |
|------|----------|
| 通用概念、跨领域术语 | `03-领域/00-概念术语/` |
| 学科专属术语 | 对应学科目录下的术语库 |
| 管理学术语 | `03-领域/50-管理学科/--管理术语表/` |
| AI 术语 | `03-领域/31-人工智能学科/00-AI术语/` |
| 资源清单 | `05-资源/` |
| 案例库 | 对应领域或项目目录 |
| Vault 整改记录 | `07-治理/Vault整改/` |

## Frontmatter 口径

术语库审计以“可检索、可筛选、可迁移”为优先目标。

| 字段 | 要求 | 说明 |
|------|------|------|
| `aliases` | 必填 | 含中文名、英文名或常见别名 |
| `tags` | 必填 | 至少含领域或术语类标签 |
| `description` | 必填 | 一句话说明术语核心含义 |
| `type` | 必填 | 固定使用 `note` |
| `create-date` | 必填 | 保留创建日期 |
| `category` | 建议 | 用于分组和 Base 排序 |
| `domain` / `field` | 建议 | 用于跨领域检索 |

`type` 字段不得混用自然语言值。术语笔记统一使用 `type: note`，术语身份由 `tags`、`aliases`、路径、`category`、`domain` / `field` 等字段表达。Base 不得再依赖 `type == "term"` 作为术语识别条件，应采用“术语目录路径 + `type == "note"` + 术语类标签/字段”的组合筛选。

## Base 审计口径

Base 审计不评价展示美观，只评价是否能可靠承载术语检索。

| 检查项 | 判定 |
|--------|------|
| `file.folder` 指向不存在路径 | P1 |
| Base 位于错误目录 | P1 或 P2，按影响范围定级 |
| 筛选条件遗漏现行术语目录 | P1 |
| Base 仍依赖 `type == "term"` | P1 |
| 字段顺序不一致 | P3 |
| 旧目录名残留但不影响结果 | P2 |

## 链接审计口径

术语链接用于建立边界、上下位、近义和实践关联。断链、歧义链接和跨库误链会降低术语库可信度。

| 问题 | 判定 |
|------|------|
| MOC 或 Base 入口断链 | P1 |
| 术语正文关键概念断链 | P2 |
| 非关键示例断链 | P3 |
| 链接指向旧工作区副本 | P2 |
| 同名多路径导致歧义 | P2 |

## 分级标准

| 等级 | 定义 | 示例 |
|------|------|------|
| P1 | 导航、数据库或目录边界失效 | 入口分裂、Base 指向不存在路径、非术语数据库混入术语 MOC |
| P2 | 检索、理解或迁移质量受损 | Frontmatter 大量空值、断链、空白术语卡 |
| P3 | 低风险不一致 | 字段顺序不一、别名不完整、分库入口展示不统一 |

## 整改验收

一项术语整改只有同时满足以下条件，才视为完成：

1. 入口唯一，MOC 可导航。
2. Base 指向实际存在的目录。
3. 术语卡能被统一筛选条件收录。
4. P1 断链清零。
5. 非术语数据库不再放置于术语目录 MOC 层。
6. 新增或修改笔记符合本规范的 Frontmatter 口径。
7. `vault-rules.yaml`、术语审计规范、术语创建 skill 与 Base 筛选条件保持一致。

## 规则引用

后续所有术语整改报告、复核记录和执行清单，均默认引用本规范作为规则源。若需变更 `type` 枚举、术语目录归属或 Base 筛选策略，应先更新本规范，再更新具体整改报告。
