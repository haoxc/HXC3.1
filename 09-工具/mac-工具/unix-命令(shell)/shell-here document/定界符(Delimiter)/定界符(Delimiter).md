## 内容
在 **在此文档(Here Document)** 的文法中，首尾两个标识符承担着完全不同的“角色”，因此它们的写法也遵循不同的规则。
简单来说：**第一个标识符负责“定义规则”，第二个标识符负责“宣告结束”。**

### 1. 起始定界符：定义解析模式 (The Declaration)

当你写 `<< 'PYEOF'` 时，你在告诉 Shell 两件事：

- **定界符名称**：这段内容的结束标记是 `PYEOF`。
- **解析规则(Quoting Rules)**：通过给标识符加上引号（单引号或双引号均可），你开启了 **字面量模式(Literal Mode)**。

> **为什么要加引号？**
> 如果不加引号（写成 `<< PYEOF`），Shell 会尝试解析 Python 代码里的特殊符号。例如，如果代码中有 `$variable`，Shell 会试图把它替换成环境变量的值。
> 加上引号后，Shell 就会“闭上眼睛”，将中间的所有内容原封不动地交给 Python，确保 Python 接收到的代码是纯净、未被修改的。

### 2. 结束定界符：纯粹的信号 (The Marker)
结尾的 `PYEOF` 必须是一个 **纯粹的、无格式的标识符**。
- **匹配原则**：Shell 在扫描输入时，只寻找与起始定义中 **去掉引号后** 的字符串完全匹配的一行。
- **语法要求**：在此文档的语法规范中，结束定界符所在的行不能有空格、不能有引号，甚至不能有注释。它必须独立成行，且字符与起始定义的名称完全一致。
---

### 3. 如果给最后一个加了引号会怎样？
如果你写成：
```bash
python3 << 'PYEOF'
# ... 你的代码 ...
'PYEOF'
```

**结果是：脚本会报错或无限等待。**
因为 Shell 仍在寻找名为 `PYEOF` 的结束标记，而它看到的 `'PYEOF'`（带引号的版本）被视为字符串内容的一部分，而不是结束信号。这就好比你在寻找一个叫“张三”的人，但对方递过来一张写着“‘张三’”的名片，系统会认为这不是同一个人。

### 总结对比

| **位置**        | **格式**       | **核心作用**                                 |
| ------------- | ------------ | ---------------------------------------- |
| **起始(Start)** | `<< 'PYEOF'` | 声明结束标记的名字，并锁定 **不进行变量替换**。               |
| **中间(Body)**  | (Python 代码)  | 被当作 **标准输入(Standard Input)** 传递给 Python。 |
| **结尾(End)**   | `PYEOF`      | 告诉 Shell：“到此为止，输入结束。”                    |
|               |              |                                          |
|               |              |                                          |

---

这种文法让你可以非常安全地在 Shell 脚本里嵌入复杂的 Python 逻辑，而不用担心 Shell 的语法逻辑干扰到 Python。
