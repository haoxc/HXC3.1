## 内容
sed (Stream Editor) 是 Unix/Linux 环境下最强大的流编辑器之一，主要用于对文本进行过滤和转换。它逐行处理文本，将当前行读入`临时缓冲区`（称为“模式空间”），执行编辑脚本，然后输出结果。

## 1. 基本语法

```bash
sed [选项] '编辑命令' 文件名
```

## 2. 常用选项

- `-n`：取消默认输出。通常与 `p` 命令配合，仅打印匹配到的行。
- `-i`：直接修改文件内容。慎用，建议带备份后缀，如 `sed -i.bak 's/old/new/g' file`。
- `-e`：执行多个编辑脚本。
- `-r` (或 `-E`)：使用扩展正则表达式 (ERE)。 

## 3. 核心编辑命令 (Subcommands) [10]

- `s` (Substitute)：最常用的替换命令。
    - `sed 's/old/new/' file`：替换每行第一个匹配项。
    - `sed 's/old/new/g' file`：全局替换行内所有匹配项。
- `d` (Delete)：删除行。
    - `sed '2d' file`：删除第 2 行。
    - `sed '/pattern/d' file`：删除包含匹配项的行。
- `p` (Print)：打印行。
    - `sed -n '1,5p' file`：打印第 1 到 5 行。
- `a` / `i` / `c`：
    - `a` (Append)：在匹配行后插入。
    - `i` (Insert)：在匹配行前插入。
    - `c` (Change)：替换整行。 [1, 4, 7, 9, 11, 12, 13, 14, 15, 16, 17, 18]

## 4. 定址方式 (Addressing)
你可以通过地址来限制命令作用的范围：
- 数字：`sed '3s/a/b/'` (只处理第 3 行)。
- 范围：`sed '1,5d'` (删除 1 到 5 行)。
- 正则：`sed '/^#/d'` (删除所有以 # 开头的注释行)。
- 特殊符号：`$` 代表最后一行。 [10, 12, 13, 16, 19]

## 5. 高阶技巧
- 多重编辑：`sed -e 's/a/b/g' -e 's/c/d/g' file`。
- 反向匹配：使用 `!`，如 `sed '/pattern/!d'` (删除不包含 pattern 的行)。
- 分段处理：`sed '10,20s/old/new/g'` (仅在 10-20 行执行替换)。 [5, 7, 12, 20, 21]
