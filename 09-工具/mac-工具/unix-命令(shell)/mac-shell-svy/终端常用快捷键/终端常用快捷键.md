## 内容
| 快捷键                   | 功能描述        | 适用场景                                    |
| --------------------- | ----------- | --------------------------------------- |
| **`Command + K`**     | **彻底清屏**    | 屏幕上有敏感 API Key 时，防止别人路过看到。              |
| **`Control + C`**     | **终止进程**    | OpenClaw 运行卡死或你想强行退出 `onboard` 模式。      |
| **`Control + L`**     | **移动光标清屏**  | 只是把当前行提到最上方，滚动条往上翻还能看到旧内容（不如 Cmd+K 安全）。 |
| **`Control + A / E`** | **跳到行首/行尾** | 修改长命令（比如修改 `apiKey` 字符串）时非常快。           |
| **`Option + Click`**  | **快速移动光标**  | 在终端的长 JSON 字符串中，按住 Option 点哪，光标就跳到哪。    |
| `Option + Backspace`  | 删除一个词       |                                         |
| `cmd+Backspace`       | 删除整行        |                                         |
|                       |             |                                         |


在 Mac 的终端（无论是自带的 Terminal、iTerm2 还是 Ghostty）中，熟练使用 Emacs 风格的快捷键可以极大地提升长命令的编辑效率。

以下是最实用的快捷键总结：

## 1. 左右快速横跳（最常用）

- `Control + A`：跳到行首（Ahead）。
- `Control + E`：跳到行尾（End）。
- `Option + ←`：向左跳过一个单词。
- `Option + →`：向右跳过一个单词。
    
    - _注意：如果 Option 键不起作用，需在终端设置中将 Option 键映射为 `Esc+` 或 `Meta`。_
    

## 2. 快速删除与剪切

- `Control + U`：剪切（删除）光标之前的所有内容。
- `Control + K`：剪切（删除）光标之后的所有内容。
- `Control + W`：删除光标前的一个单词。
- `Control + Y`：粘贴（召回）刚刚删除/剪切的内容。

## 3. 高阶编辑技巧

- `Control + T`：交换光标前后的两个字符。
- `Control + L`：清屏（效果等同于 `clear`，但更快）。
- `Control + _`：撤销上一步编辑（Undo）。

---

## 4. 终极杀招：在编辑器里写长命令

如果你面对的是一个几十行的超级长命令（比如复杂的 Docker 命令或 `uv` 脚本），用快捷键跳来跳去也很累。

- 快捷键：`Control + X` 接着按 `Control + E`
- 功能：它会立刻把当前的命令丢进一个临时的 Vim（或默认编辑器）里。
- 操作：在编辑器里改爽了之后，保存退出（`:wq`），命令会自动在终端执行。

## 5. 搜索历史命令

- `Control + R`：进入搜索模式，输入关键词即可找回以前输入过的超长命令，按回车直接执行，按左右键进入编辑。

---

配置建议：  
如果你使用的是 iTerm2，建议在 `Settings -> Profiles -> Keys -> Key Bookmarks` 中选择 Natural Text Editing 预设，这样 `Command + Backspace`（删除整行）等 Mac 原生习惯也能在终端生效。
