## 万能公式
**“万能公式”**。只要套入这个模板，你会发现各种复杂的工具（Docker, Git, AWS CLI, ls）逻辑都是通行的。 
![[2eb36a982358ad273f0a176c60ff81bf_MD5.jpg]]

## 三大组件

### 1. 命令 (Command) —— **“动词：你要做什么？”** 
- 这是程序的名称，代表你要发起的动作。
- _例子：_ `ls` (列出), `rm` (删除), `docker` (调用容器引擎)。 
### 2. 选项/标志 (Options/Flags) —— **“副词：怎么做？”** 
- 用来修改命令的行为。
	- **短格式**：单个减号 `-` 加字母（如 `-i`, `-t`），多个短选项通常可以合并，如 `-it`。
	- **长格式**：双减号 `--` 加单词（如 `--rm`, `--entrypoint`），可读性更高。
- _记忆点：_ 它是用来**开关功能**或**调整精度**的。 
### 3. 参数 (Arguments) —— **“宾语：对谁做？”** 
- 命令操作的对象。
- _例子：_ 文件名、文件夹路径、镜像名称（如 `node:24-alpine`）。

### 示例
Dock例子套用公式： 

| 部分           | 内容                         | 角色   | 含义              |
| ------------ | -------------------------- | ---- | --------------- |
| **Command**  | `docker run`               | 核心动词 | 运行容器            |
| **Options**  | `-it --rm --entrypoint sh` | 行为修饰 | 交互式、运行完删掉、把入口改掉 |
| **Argument** | `node:24-alpine`           | 操作对象 | 针对这个特定的镜像       |

## 进阶：Unix 哲学的“管道”公式 

如果你想把多个命令串起来，公式会进化为：
**`命令1 | 命令2 | 命令3`** 
- **`|` (Pipe)**：把前一个命令的“输出”变成后一个命令的“输入”。
- _例子：_ `docker ps -a | grep node` (列出所有容器，然后从中筛选出包含 node 的行)。 

> 记忆小贴士： 
- **空格是灵魂**：Unix 命令依靠空格来区分这三个部分，多一个少一个都不行。
- **顺序很重要**：通常遵循 `动词 -> 副词 -> 宾语` 的逻辑。
- **查阅字典**：忘记某个命令的具体选项时，直接在后面加 `--help`，例如 `docker run --help`。 
你可以参考 GNU 命令行标准指南 深入理解这种`设计美学`。
