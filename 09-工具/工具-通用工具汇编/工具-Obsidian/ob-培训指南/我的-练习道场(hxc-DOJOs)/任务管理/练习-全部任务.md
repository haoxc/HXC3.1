```
#### 2. 代码块解释

*   `not done`：这是 `Tasks` 插件的查询过滤器。它会筛选出所有**未完成**的任务（即 `[-]` 或 `[ ]` 状态）。
*   如果您希望搜索所有任务，包括已完成的任务，可以将查询更改为：
    ```
    ```tasks
    any status
    ```
    ```
#### 3. 如何使用

*   您可以在您的主页（Dashboard）笔记中创建一个这样的查询代码块，这样每次打开主页时，都可以一目了然地看到所有未完成的任务。
*   您也可以创建多个不同的查询代码块，例如：
    *   **未完成的任务（按截止日期排序）**
        ```
        ```tasks
        not done
        sort by due
        ```
        ```
    *   **高优先级的任务**
        ```
        ```tasks
        not done
        priority is high
        ```
        ```
```
