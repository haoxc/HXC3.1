## 语法三要素
符合人类直觉的语言学结构：**“对象 - 关系 - 容器”**。
### A. 对象与关系 (Objects & Connections)
这是 D2 的“主谓宾”结构。
- **语法：** `主语 -> 宾语 : 谓语(标签)`
- **逻辑：** 定义拓扑结构。
- **代码示例：**
```
User -> Server: 发送请求  # User 是主语，Server 是宾语
Server -> Database: 查询数据
```

### ### B. 容器与嵌套 (Containers & Nesting)

这是 D2 的“层级逻辑”。通过花括号 `{}` 建立父子关系。
- **逻辑：** 定义系统的**作用域 (Scope)** 和 **模块化 (Modularity)**。
- **代码示例：**
```
Cloud_System {   # 父级容器
  Server         # 子对象
  Database       # 子对象
  Server -> Database # 内部逻辑
}
```

### C. 属性与修饰 (Attributes & Styling)

这是 D2 的“形容词”和“副词”。用于改变对象的存在形式（颜色、形状、图标）。
- **逻辑：** 不改变拓扑结构，只改变**视觉语义**。
- **代码示例：**
```
Server: {
  shape: cylinder        # 定义形状（存在形式）
  style: {
    fill: "#f4f4f4"      # 定义外观
    stroke: "#333"
  }
}
```

## 3. 核心引擎：布局逻辑 (Layout Engines)
**布局引擎**是 D2 的“大脑”，它决定了逻辑关系如何转化为物理图像。

| **引擎名称**             | **逻辑特点**                | **适用场景**                     |
| -------------------- | ----------------------- | ---------------------------- |
| **dagre** (默认)       | **层级导向**。自上而下，像家谱图。     | 简单的流程图、状态机。                  |
| **elk**              | **算法导向**。追求连线的最短路径和对称性。 | 复杂的网络拓扑、电路设计（你刚才的物联网图适合用这个）。 |
| **sequence_diagram** | **时间导向**。强制转换为时序逻辑。     | 交互流程、生命周期分析。                 |

## 文法要点
- **定义一个点：** `Name`
- **定义一条线：** `Name1 -> Name2`
- **定义一个组：** `Group { ... }`
- **加点解释：** `Name.description: "这是解释"`
- **换个样子：** `Name.shape: square`
- **导入图标：** `Name.icon: material/cloud`
