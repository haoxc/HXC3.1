# 📝 知识卡片：D2 语言 Shape 类型全集

## 1. 基础几何类 (General Logic)

> [!abstract] 用途
>
> 用于表达最通用的概念、实体或分组。

| **Shape 关键词**          | **逻辑含义**        | **适用场景**        |
| ---------------------- | --------------- | --------------- |
| **(默认)** / `rectangle` | **实体 (Entity)** | 任何通用的对象、步骤。     |
| `square`               | **正方实体**        | 同上，视觉上更紧凑。      |
| `circle`               | **节点 (Node)**   | 状态机中的状态、拓扑图中的点。 |
| `oval`                 | **开始/结束**       | 流程图的起止点。        |
| `text`                 | **纯文本**         | 注释、标签、无边框的说明。   |

---

## 2. IT 架构类 (Technical Semantics)

> [!abstract] 用途
>
> 只有在描述计算机系统架构时使用，具有公认的行业语义。

| **Shape 关键词** | **视觉隐喻**  | **逻辑含义**                             |
| ------------- | --------- | ------------------------------------ |
| `cylinder`    | **圆柱体**   | **存储** (Database, Disk, Storage)。    |
| `cloud`       | **云朵**    | **网络/外部环境** (Internet, VPC)。         |
| `queue`       | **队列**    | **缓冲/管道** (Kafka, RabbitMQ, Stack)。  |
| `package`     | **文件夹/包** | **模块/容器** (Java Package, Go Module)。 |
| `server`      | **服务器**   | **算力单元** (VM, Physical Machine)。     |

---

## 3. 流程控制类 (Flow Logic)

> [!abstract] 用途
>
> 专门用于构建算法逻辑或业务流程图。

| **Shape 关键词**   | **视觉隐喻**  | **逻辑含义**                     |
| --------------- | --------- | ---------------------------- |
| `diamond`       | **菱形**    | **判断/分支** (If-Else, Switch)。 |
| `parallelogram` | **平行四边形** | **输入/输出** (I/O, Read/Write)。 |
| `hexagon`       | **六边形**   | **准备/初始化**。                  |
| `step`          | **梯形**    | **手动操作**。                    |

---

## 4. 特殊模式类 (Meta Modes)

> [!important] 逻辑突变
>
> 这些 shape 不仅仅改变形状，它们会**改变整个图的渲染逻辑**或**内部结构**。

| **Shape 关键词**      | **逻辑变化**                                |
| ------------------ | --------------------------------------- |
| `sequence_diagram` | **时序模式**。将整个画布变为垂直时间轴逻辑（泳道图）。           |
| `class`            | **类图模式**。允许定义 Field (字段) 和 Method (方法)。 |
| `sql_table`        | **ER图模式**。允许定义列名、类型、约束。                 |
| `code`             | **代码块模式**。用于在图中直接展示代码片段。                |
| `image`            | **图片模式**。引用外部 URL 图片。                   |
|                    |                                         |

## 🚀 实践指南：代码速查演示

把这段代码复制到 D2 中，你就能直观看到不同“存在形式”的区别：

代码段

```
direction: right

# 1. 基础实体
普通节点: { shape: rectangle }
开始状态: { shape: oval }

# 2. 逻辑判断
是否通过: { shape: diamond }

# 3. IT 组件
数据库: { shape: cylinder }
互联网: { shape: cloud }
消息队列: { shape: queue }

# 4. 角色
管理员: { shape: person }

# 关系连接
开始状态 -> 管理员: 登录
管理员 -> 是否通过: 验证权限
是否通过 -> 消息队列: 权限(True)
是否通过 -> 互联网: 拒绝(False)
消息队列 -> 数据库: 持久化
```

## 💡 教练建议 (Coach's Tip)

在使用 `shape` 时，请遵循**“`形式追随功能 (Form follows Function)`”** 的原则：

1. **不要为了好看而乱用：** 不要把“数据库”画成 `cloud`，这会导致技术语义混淆（逻辑谬误）。
2. **默认即最优：** 如果你不确定该用什么，就用默认的（矩形）。
3. **一致性：** 在同一张图中，同类事物必须保持相同的 `shape`（例如：所有的微服务都统一用 `package`）。
