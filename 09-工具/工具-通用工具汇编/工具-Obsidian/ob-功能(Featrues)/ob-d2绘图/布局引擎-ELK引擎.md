Eclipse Layout Kernel (ELK) 是一个为图表编辑器和查看器提供自动布局算法和支持基础设施的工具。

## 思想
- ELK（Eclipse Layout Kernel）的核心思想是提供一个可扩展的框架，用于将图编辑器或查看器连接到各种自动布局算法，并通过一系列可配置的选项来精细控制布局结果。
- 其最主要的算法是基于`层次`的布局方法，特别适用于`流程图和有向无环图` (DAGs)。
# 摘要

Eclipse Layout Kernel (ELK) 旨在简化图表的布局过程，它提供了一系列自动布局算法，并提供了一个连接这些算法与图表编辑器和查看器的基础设施。ELK 不直接渲染图表，而是计算图表元素的位置（以及可能的尺寸）。该网站展示了一个使用 ELK 生成的布局示例，该示例展示了 ELK 的一些特殊功能，如端口（即边缘连接到节点边界的点）和层次节点（即包含子节点的节点）。此外，网站还提到了三个使用 ELK 自动布局功能的应用程序：KlassViz（用于在 Eclipse 中实时生成类图）、DebuKViz（用于在调试 Java 程序时生成变量的可视化）以及 Ptolemy II（用于通过自动布局使图形建模变得更加容易）。

## 思想
- **层次布局**：其旗舰算法是基于 [[Sugiyama算法]] 等人提出的分层方法。该方法的核心在于：
    - **强调方向**：通过将节点排列在不同的层（或层次）中，使尽可能多的边指向同一方向（通常是自上而下或自左而右），从而清晰地展示流程或依赖关系。
    - **减少交叉**：在分层的基础上，算法会重新排列层内节点的顺序，以**最小化边之间的交叉**，提高图的可读性。
    - **计算坐标**：最后，计算节点和边的具体坐标，包括正交布线或样条曲线布线，并尊重端口约束（边在节点边界上的特定连接点）。
