## 内容
在 Node.js 中，“全局”（Global）==主要指包的安装位置和使用范围==。

当你使用 `npm install -g <包名>` 命令时，就是在进行全局安装。 [1]

## 1. 核心区别：全局 vs 局部

| 特性   | 全局安装 (`-g`)                                              | 局部/本地安装 (默认)                         |
| ---- | -------------------------------------------------------- | ------------------------------------ |
| 存放位置 | 系统公共目录（如 Windows 的 `AppData\npm` 或 Linux 的 `/usr/local`） | 当前项目的 `node_modules` 文件夹中            |
| 主要用途 | 命令行工具（CLI）。让你能在终端任何地方直接运行命令                              | 项目依赖。供项目代码通过 `import` 或 `require` 调用 |
| 复用性  | 整个操作系统通用                                                 | 仅在当前项目目录下有效                          |
| 版本管理 | 全系统只有一个版本，升级可能导致旧项目崩溃                                    | 每个项目可以拥有独立且不同版本的包                    |
|      |                                                          |                                      |
## 2. 什么时候该用“全局”？

只有当你需要把一个包当作独立的软件/工具在终端使用时，才考虑全局安装。
- 常见全局包示例：
    - `nodemon`：自动重启项目的工具。
    - `typescript`：提供 `tsc` 编译命令。
    - `hexo-cli`：博客搭建工具。 [6, 12]

## 3. 注意事项
- 代码引入问题：默认情况下，你的项目代码无法直接引用全局安装的包。如果你在代码里写 `import`，TypeScript/Node 会去本地 `node_modules` 找，找不到就会报错。
- 最佳实践：现代开发中，建议尽量使用局部安装。即使是命令行工具，也可以局部安装后通过 `npx <命令>` 来运行，这样能避免不同项目间的版本冲突。
