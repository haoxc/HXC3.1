## 内容

| **特性**   | **矢量数据 (Vector)**                      | **栅格数据 (Raster)**                |
| -------- | -------------------------------------- | -------------------------------- |
| **基本单元** | 点 (Point)、线 (Line)、面 (Polygon)         | 像素 (Pixel) / 网格 (Grid)           |
| **表现形式** | 数学几何图形，边界清晰                            | 连续的图像矩阵，边界模糊                     |
| **常见用途** | 行政区划、道路网络、POI兴趣点                       | 卫星影像、高程图 (DEM)、气象分布              |
| **文件格式** | `.shp` (Shapefile), `.geojson`, `.kml` | `.tif` (GeoTIFF), `.jpg`, `.png` |
| **优势**   | 数据量小，适合网络分析和精确查询                       | 适合表现连续变化的现象（如温度、海拔）              |
|          |                                        |                                  |

地理空间（Geospatial）数据主要分为 **矢量数据**（Vector）和 **栅格数据**（Raster）两大类。不同的格式对应不同的应用场景。

### 1. 矢量数据 (Vector Data)
矢量数据通过坐标点（Point）、线（Line）和多边形（Polygon）来定义地理实体，通常用于表示离散且有明确边界的对象。 ![UC Irvine](https://encrypted-tbn0.gstatic.com/faviconV2?url=https://guides.lib.uci.edu&client=AIM&size=128&type=FAVICON&fallback_opts=TYPE,SIZE,URL)UC Irvine

| 常见格式               | 全称                        | 目的与用途                                                      |
| ------------------ | ------------------------- | ---------------------------------------------------------- |
| **Shapefile**      | ESRI Shapefile (`.shp`)   | 行业标准格式，兼容性最广。用于存储行政边界、道路网、设施点等。                            |
| **GeoJSON**        | Geographic JSON (`.json`) | **Web 应用**的首选。结构简单，易于被 JavaScript 和 GitHub 等平台解析。          |
| **KML/KMZ**        | Keyhole Markup Language   | **Google Earth** 专用格式。常用于简单的地图展示、航迹记录和 3D 可视化。             |
| **[[GeoPackage]]** | OGC GeoPackage (`.gpkg`)  | 现代化的、基于 SQLite 的单文件数据库。旨在取代 Shapefile，支持在一个文件中存储多层矢量和栅格数据。 |
| **GML**            | Geography Markup Language | 基于 XML 的标准，主要用于不同系统间的地理数据交换和互操作。                           |

### 2. 栅格数据 (Raster Data)

栅格数据由`像素网格（`Pixels）组成，每个像素带有一个属性值，适合表示`连续变化`的自然现象。

| 常见格式           | 全称                                | 目的与用途                                                    |
| -------------- | --------------------------------- | -------------------------------------------------------- |
| **GeoTIFF**    | Tagged Image File Format (`.tif`) | **遥感影像**和卫星图片的最常用格式。将地理坐标嵌入图片，常用于地图底图和分析。                |
| **ASCII Grid** | Arc/Info ASCII Grid (`.asc`)      | 以文本形式存储的网格数据。因易读性好，常用于科研领域的**降雨量**、**气温**或**高程（DEM）**数据。 |
| **NetCDF**     | Network Common Data Form          | 多维科学数据格式。常用于气象、海洋等**时间序列**数据（例如：某海域过去 10 年的水温变化）。        |
| **HDF**        | Hierarchical Data Format          | 存储大规模卫星数据包。能够同时包含图像、元数据和多层光谱信息。                          |
|                |                                   |                                                          |

### 3. 特殊用途格式

- **LAS/LAZ**: 专门用于存储 **LiDAR（激光雷达）** 点云数据，用于创建高精度的 3D 地形模型。
- **PostGIS**: 一种空间数据库插件（基于 PostgreSQL），用于在企业级环境中管理、查询和分析海量地理数据。
- **MBTiles**: 瓦片地图（Map Tiles）格式，用于在移动端或离线地图应用中快速渲染地图底图。
