## 内容
- [[同时在线]] (用户): **衡量系统 “能挂住多少人”**，为**容量、资源、稳定性**服务 (用户规模、连接数)
- 吞吐量 (系统): 系统每秒钟能处理多少东西(能干多少活)
	- *关注*
		- 系统`处理能力, 忙不忙`
	- *本质*
		- 系统处理效率
	- *单位*
		-  [[每秒查询 (QPS)]]
		- [[每秒事务 (TPS)]]
- [[并发数]]: **看 “能扛多少事”**。
- [[响应时间 (RT)]]
	- *关注*: 系统反应快不快
	- *本质*: 单位请求的处理速度
	- *单位*: 毫秒 (ms)
	- *期望*: **越小体验越好**
- 线程数

### 对比

| 指标   | 关注核心       | 通俗理解（餐厅）  |
| ---- | ---------- | --------- |
| 同时在线 | 能挂多少人      | 店里坐了多少人   |
| 并发   | 同一时间多少人在操作 | 同时点菜的人    |
| 吞吐量  | 每秒能处理多少请求  | 厨房每秒出多少菜  |
| 响应时间 | 单次请求耗时多久   | 点菜到上菜用了多久 |
