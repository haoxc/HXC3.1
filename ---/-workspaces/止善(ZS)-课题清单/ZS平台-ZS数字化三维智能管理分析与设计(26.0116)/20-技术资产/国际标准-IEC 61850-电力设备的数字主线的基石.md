## 内容

> [!WARNING]
> 电力设备的 **伴随模型** 不是一个全新的发明，而是对成熟的 **IEC 61850** 标准的 **OPC UA** 封装和映射

在电力设备的 **数字主线** 中， IEC 61850 是基石。
它定义了变电站自动化和电力设备（如断路器、变压器、发电机）的 **逻辑节点（Logical Nodes, LN）** 和 **属性模型** 。 
- **本质** ： IEC 61850 是一套完整的电力设备描述语言（使用 **SCL** 配置文件），确保了不同厂家生产的智能电子设备（IED）可以互操作。
- **挑战** ：它主要专注于电力系统内部，协议较为封闭，难以与上层 **ERP** 、 **维护系统** 或 **云平台** 直接集成。

### 1. 核心本质：从“数据传输”转向“语义建模”
传统的电力协议（如 Modbus）只传输无意义的寄存器地址（如 `Address 4001 = 1`），而 IEC 61850 引入了**面向对象**的建模：
- **逻辑节点 (Logical Nodes, LN)**：这是其灵魂。将设备功能拆解为标准化的最小单元。
    - `XCBR`：代表**断路器**（Circuit Breaker）。
    - `TCTR`：代表**电流互感器**（Current Transformer）。
    - `PTUV`：代表**低电压保护**（Undervoltage Protection）。
- **自描述性**：设备通过网络直接告诉监控系统：“我是 A 厂的断路器，我的当前位置（Pos）是‘合闸’”。这种**语义一致性**是构建电力数字主线的基石。

### 2. 三大关键通信机制

针对复杂电力系统不同的实时性需求，它定义了三种映射：
- **MMS (Manufacturing Message Specification)**：用于`站控层`。负责传输非实时的报表、历史数据和控制指令。
- **GOOSE (Generic Object Oriented Substation Event)**：用于`间隔层`。极其高效的点对点传输（毫秒级），专门用于**保护跳闸**和状态联锁。
- **SV/SMV (Sampled Values)**：用于`过程层`。将电压、电流的瞬时采样波形数字化，支持**数字采样**的高频传输。

### 3. 配置语言：SCL (Substation Configuration Language) 

IEC 61850 定义了一套基于 **XML** 的配置语言（IEC 61850-6），形成了电力系统的**数字资产基准**： 
- **ICD (Capability Description)**：设备能力文件，由厂家提供。
- **SCD (Substation Configuration Description)**：全站配置文件。它描述了整个变电站的拓扑、逻辑连接和通信参数。
- **数字主线价值**：SCD 文件就是变电站的**静态数字孪生**，记录了系统设计的“初始真相”。

### 权威出处
- **标准组织**：IEC TC 57（电力系统控制及其通信委员会）。
- **核心文档**：IEC 61850-7-4 (模型定义) 与 IEC 61850-6 (配置语言)
