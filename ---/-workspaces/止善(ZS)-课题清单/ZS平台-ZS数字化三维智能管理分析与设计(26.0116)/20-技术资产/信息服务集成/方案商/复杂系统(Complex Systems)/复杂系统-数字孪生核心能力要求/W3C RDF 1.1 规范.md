## 内容

在 **数字主线** 的底层架构中， **W3C RDF 1.1** （Resource Description Framework）是所有数据互操作的“原子级”规范。它不仅是一个文件格式，更是一套 **语义建模语言** ，负责定义 **复杂系统** 中万物之间的关系。

### . 核心模型：三元组 (Triple)

**RDF** 的一切表达都由“ **主体 (Subject)** — **谓体 (Predicate)** — **客体 (Object)** ”组成。
- **示例** ： `[卫星 A]` — `[具有重量]` — `[500kg]`。
- **作用** ：这种结构将碎片化的数据转变为 **有向图 (Directed Graph)** 。在 **复杂系统** 中，无论数据来自 **CAD** 还是 **Excel** ，只要转化为三元组，就能拼接成一张巨大的 **知识图谱** 。

### 2. 标识符： **URI** (Uniform Resource Identifier)

在 **RDF 1.1** 中，主体和谓体必须由 URI 标识。
- **规范要求** ：所有的关系定义（谓体）都要有明确的命名空间。例如，使用 `oslc_rm:satisfiedBy` 来表示“被验证”这一语义。
- **价值** ：解决了 **复杂产品** 研发中常见的“同名异义”问题。全球不同供应商定义的“压力”属性，通过不同的 URI 实现精准区分。 [出处：W3C RDF 1.1 Concepts and Abstract Syntax]

### 3. 多种序列化格式 (Serialization)

**RDF 1.1** 规范定义了数据如何被转换成文本以便在 **数字主线** 中传输：
- **Turtle (.ttl)** ：最易读的纯文本格式，适合开发人员调试。
- **JSON-LD** ：最前沿的格式，将 **语义数据** 嵌入 **JSON** 中，是 **Agent** 模式和 **Web** 开发的首选。
- **RDF/XML** ： **OSLC** 早期最常用的格式，具有极强的严谨性。 [出处：W3C RDF 1.1 XML Syntax]

### 4.  核心概念：字面量 (Literals) 与 **空节点** (Blank Nodes)

- **字面量** ：用于存储具体数值（如 `3.14`）或字符串，并支持 **数据类型 (Datatypes)** 标注（如 `xsd:float`）。
- **空节点** ：用于描述那些没有唯一标识但有结构的中间对象（如一个临时的测试组合）。 [出处：W3C RDF 1.1 Semantics]

### 5. 在数字主线中的解析逻辑

当仿真软件解析需求的 **URI** 时，它实际上是在执行以下 **RDF** 动作：
1. **图合并 (Graph Merging)** ：将需求服务器返回的 **RDF 图** 与本地的仿真数据图合并。
2. **语义查询 (SPARQL)** ：使用 **SPARQL** 语言在合并后的图中搜索：“找出所有未通过测试的 **关键需求** ”。

### **权威出处** 
- **标准定义** ： W3C RDF 1.1 官方规范系列
- **关联标准** ： **OWL** (Web Ontology Language)，用于定义更复杂的 **逻辑推理** 规则。

### **总结** 
**RDF 1.1** 是 **复杂系统** 数字主线的“ **语法书** ”。它确保了不同工具产生的每一个数据点都能被结构化地连接在一起，从而实现自动化的 **追溯** 和 **推理** 。
