## 内容
电力系统安全防护遵循 **"安全分区、网络专用、横向隔离、纵向认证"** 的十六字方针。

### 1. 核心分级与分区要求

根据 **国家发改委第 27 号令**（2025 年 1 月 1 日起施行），`电力监控系统`需实施分区防护：

| 防护区域               | 安全等级   | 业务范例                  | 隔离要求                      |
| ------------------ | ------ | --------------------- | ------------------------- |
| **生产控制大区（安全 Ⅰ 区）** | **最高** | 实时调度、[[继电保护]]、配电自动化控制 | 与非控制区物理隔离，严禁互联网连接         |
| **生产控制大区（安全 Ⅱ 区）** | **高**  | 调度员培训、水调自动化、电能量计量     | 与 Ⅰ 区逻辑隔离，与 Ⅲ 区横向单向隔离     |
| **管理信息大区（安全 Ⅲ 区）** | **中**  | 办公自动化（OA）、生产管理系统      | 采用**电力专用单向安全隔离装置**与 Ⅱ 区连接 |
| **管理信息大区（安全 Ⅳ 区）** | **低**  | 营销系统、外网互联业务           | 与 Ⅲ 区逻辑隔离，严禁直接连接生产区       |
|                    |        |                       |                           |

### 2. 关键技术防护要求 (详细)

在等保 2.0 的 **安全通用要求** 基础上，电力监控系统有以下刚性技术要求：
- **横向隔离 (Horizontal Isolation)**：
    - 生产控制大区与管理信息大区之间必须部署**电力专用横向单向安全隔离装置**。
    - 该装置需具备物理层断开功能，确保数据只能单向传输，彻底防止病毒从办公网渗透至控制网。
- **纵向认证 (Vertical Authentication)**：
    - 在各级调度中心、厂站之间建立通信时，必须部署**电力专用纵向加密认证装置**。
    - 利用**电力调度数字证书**实现身份双向认证、数据加密及完整性保护。
- **恶意代码防范**：
    - 生产控制大区禁止使用通用的联网杀毒软件实时更新。
    - 必须采用**经测试验证过的特征码**手动更新，或使用具备防病毒能力的加固主机。
- **计算环境加固**：
    - **身份鉴别**：重要系统应采用**双因素身份认证**（如口令+数字证书）。
    - **可信验证**：基于可信根对系统引导程序、应用程序进行可信度量，防止系统被篡改。
