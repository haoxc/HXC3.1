# 课题知识转入领域知识的判定标准

**结论**：课题层知识转入知识层，不是“搬文件”，而是“提炼稳定结论”。课题层保留过程、材料、证据和争议；知识层只接收可复用、可解释、可维护的稳定知识。

## 两层分工

| 层级  | 管什么                       | 不管什么      |
| --- | ------------------------- | --------- |
| 课题层 | 问题、过程、原始材料、证据、案例、争议、阶段性判断 | 不追求最终稳定表达 |
| 知识层 | 稳定概念、方法、模型、原则、分类、可复用结论    | 不保存研究全过程  |

## 转入条件

一条知识从课题层进入知识层，至少同时满足 5 个条件：

| 条件    | 判断问题               | 通过标准                         |
| ----- | ------------------ | ---------------------------- |
| 稳定性   | 这个判断是否已经不再频繁变化？    | 连续多次讨论或资料验证后，核心结论没有明显反转      |
| 泛化性   | 它是否超出当前课题，能用于其他场景？ | 能解释一类问题，而不是只服务一次任务           |
| 可复用性  | 未来是否会被反复引用？        | 能成为术语、方法、模型、原则、模板或检查项        |
| 证据充分性 | 它是否有来源、案例或推理链支撑？   | 至少能说明“为什么这样判断”               |
| 归口清晰  | 它应该由哪个主页面负责解释？     | 能指定一个 canonical 页面，其他页面只链接引用 |

## 不应转入的情况

以下内容留在课题层：

- 原始资料、摘录、会议记录、调研过程。
- 尚未验证的灵感、假设、临时判断。
- 只对单个项目、单次任务有效的操作记录。
- 多个结论仍在冲突，尚未完成辨析。
- 需要保留完整上下文才能理解的长案例。

## 转入形态

| 方式 | 适用情况 | 处理 |
| --- | --- | --- |
| 提炼成新页 | 出现新的稳定概念、模型、方法 | 在 `03-领域/` 下创建主解释页 |
| 合并到旧页 | 已有 canonical 页面 | 更新旧页，并在课题页保留来源链接 |
| 只做索引链接 | 内容仍依赖课题上下文 | 知识层 MOC 链到课题页，不复制正文 |

## 判定流程

1. 先问：这是不是可复用知识，还是一次性过程材料？
2. 再问：它能否脱离原课题被理解？
3. 再问：是否已有主解释页可以承接？
4. 再问：是否存在未解决争议或反例？
5. 最后决定：新建、合并、只链接，或继续留在课题层。

## 建议评分

每项 0-2 分，总分 10 分：

| 维度 | 0 分 | 1 分 | 2 分 |
| --- | --- | --- | --- |
| 稳定性 | 仍在变化 | 基本稳定 | 多次验证稳定 |
| 泛化性 | 只适合当前任务 | 可用于相近场景 | 可解释一类问题 |
| 可复用性 | 很少再用 | 偶尔引用 | 高频引用或可模板化 |
| 证据充分性 | 无依据 | 有经验判断 | 有来源、案例或推理链 |
| 归口清晰 | 不知放哪 | 大致可归类 | canonical 页面明确 |

判定建议：

- `0-4 分`：留在课题层。
- `5-7 分`：暂不迁移，先在知识层 MOC 做索引链接。
- `8-10 分`：提炼或合并进入知识层。

## 例子

| 课题层内容 | 是否转入 | 目标形态 |
| --- | --- | --- |
| “管理学科目录当前有 31 个未解析 wikilink” | 否 | 留在课题审计记录 |
| “MOC 是地图，不是正文” | 是 | 进入 MOC 设计原则 |
| “领域层用 HXC，课题层用 llm-wiki” | 是 | 进入知识内容治理原则 |
| “经营维度有重复解释风险” | 部分 | 作为案例留课题层，原则进入知识层 |

## 收口规则

- 转入知识层时，只转结论、模型、原则，不转全过程。
- 课题层保留来源和演化痕迹。
- 知识层必须能被未来读者独立理解。
- 任何转入都要指定 canonical 页面，避免一棵树多张嘴。
