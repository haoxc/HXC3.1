# 管理学科目录结构评估

**结论**：`03-领域/50-管理学科` 的方向不冲突，但执行上已经出现局部冲突。它按“术语 / 摘要 / 课题 / 工具 / 知识”组织管理学科，适合该领域的复杂性；问题在于入口 MOC 弱、MOC 层级分裂、课题与知识混放、同一概念多处解释。

## 现有组织逻辑

| 模块 | 当前角色 | 判断 |
| --- | --- | --- |
| `--管理术语表` | 管理术语、辨析、术语数据库 | 合理，但需与全局术语库分工 |
| `01-管理-摘要` | 概念辨析、压缩版理解 | 合理，但建议改成“辨析与摘要” |
| `02-管理-课题` | 战略指标量化、持续改善、项目管理 | 合理，但不宜承载深度研究全过程 |
| `03-管理-工具` | FMEA、帕拉图、BSC、OKR、鱼骨图 | 合理，但建议改成“方法工具” |
| `04-管理-知识` | 战略规划、经营管理、绩效管理 | 合理，应承接稳定沉淀内容 |

## 与核心思想的冲突点

### 入口 MOC 弱

`[[50-管理学科]]` 当前更像空壳笔记，不像领域 MOC。它应该承担该领域的一级导航，而不是只保留 `## 内容` 空列表。

### MOC 层级分裂

`--/MOC/` 形成了第二套导航系统，容易和 folder note MOC 竞争。HXC Vault 的规则更偏向“文件夹笔记即 MOC”，MOC 是地图，不应成为独立内容命名空间。

### 课题与知识混放

`课题-指标设计` 处在 `04-管理-知识/绩效管理/` 下，而“战略指标量化”处在 `02-管理-课题/` 下。后续新增内容时，容易无法判断该按“研究课题”归档，还是按“稳定知识”归档。

### 一棵树多张嘴

`经营维度` 同时出现在术语表和知识目录。两类页面可以共存，但必须明确分工：术语页只负责定义，知识页只负责模型、应用和案例。

### 链接卫生风险

轻量检查显示，排除 `.claude/worktrees` 后，`50-管理学科` 下 84 个 Markdown 文件中仍有 31 个未解析 wikilink、1 个歧义链接。它们未必全部是 P1 问题，但已经影响导航可信度。

## 判断

当前结构不是原则错误，而是“领域沉淀层”和“课题研究层”逐渐混在一起。解决重点不是大搬家，而是重建入口、明确边界、指定概念主解释页。
