# 术语规范整改计划

**来源报告**：[[26.0502-术语规范审计报告]]
**整改对象**：术语审计规范、术语 Base、术语卡 Frontmatter
**整改目标**：统一术语笔记 `type: note` 口径，恢复 Base 可筛选性，消除旧路径和规则源冲突。

## 整改原则

- 以 `99-设置/vault-rules.yaml` 的 `terminology.frontmatter_type: note` 为声明层。
- 术语身份不再依赖 `type: term`，改由路径、标签、别名和必要字段表达。
- Base 的 `全部(All)` 视图必须加路径约束，避免误收全库普通 `type: note` 笔记。
- 先消除 P1 规则冲突，再处理 P2 元数据质量。

## 任务清单

| ID | 来源问题 | 整改动作 | 状态 |
| --- | --- | --- | --- |
| R-001 | `type` 字段口径冲突 | 将术语审计规范固化为 `type: note` | 已完成 |
| R-002 | Base 依赖 `type == "term"` | 将术语相关 Base 改为 `type == "note"` | 已完成 |

| R-006 | 验收标准缺规则源同步 | 在术语审计规范中补充规则源一致性验收项 | 已完成 |

## 验收标准

- `03-领域` 下 Base 不再出现 `type == "term"`。
- 术语相关 Base 不再指向旧 `---/` 数据库看板路径。
- 术语目录内不再出现 `type: term`、`type: 术语卡片`、`type: 概念定义` 或空 `type`。
- 术语审计规范明确 `type: note`，不再保留二选一口径。

## 后续注意

现有术语卡仍可能缺 `aliases`、`tags` 或 `description`。本轮优先完成规则口径整改，不展开逐卡内容质量补齐。
