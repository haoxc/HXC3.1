# Codex Skill Hermes 审计整改

## 变更背景

用户要求根据 Hermes 的审计规则整改 Codex skill。审计发现 Codex 运行版虽与生成版 `SKILL.md` 一致，但同步脚本只同步 `SKILL.md`，没有同步 `agents/` 与 `references/`，导致 Codex 运行时可能继续使用旧版 UI 元数据或旧版规则引用。

同时，`note-crafter` 仍将课题创建路由到 `-workspaces/`，与用户对 `---` 的临时工作区定位存在张力。经用户纠正，不引入 `08-课题` 这个一级目录；课题创建必须先确认承载目录。

## 变更内容

- `99-设置/vault-rules.yaml`
  - 未新增 `08-课题`。
  - 将 `---` 描述调整为临时工作区。
  - 修正 SVG 豁免路径为 `---/-workspaces`。
- `99-设置/agent-skills/note-crafter/core.md`
  - 移除课题创建对固定一级目录的默认假设。
  - 明确课题创建前必须确认承载目录；稳定知识回流 `03-领域/`，规则决策回流 `07-治理/`。
- `99-设置/agent-skills/note-crafter/manifest.yaml`
  - 修正 Codex/Hermes prepend 中损坏的路由表行。
  - 将课题关键词路由改为“先确认承载目录”。
- `99-设置/agent-skills/note-crafter/agents/openai.yaml`
  - 将 Codex runtime 已存在的 UI 元数据纳入源目录。
- `99-设置/agent-skills/pjx-common-auditor/manifest.yaml`
  - 修正 `ignores` 中的临时工作区路径。
- `99-设置/agent-skills/sync-agent-skills.py`
  - 新增 `agents/`、`references/`、`scripts/`、`assets/` 资源目录同步。

## 设计决策

- 不直接手改 `~/.codex/skills/*` 作为唯一变更点，避免运行时漂移。
- 以 `99-设置/agent-skills/*` 为源，再同步到 Codex runtime。
- 不把 `---/-workspaces` 继续强化为正式课题层；它只保留临时工作区身份。
- 不新增 `08-课题`；课题一级承载结构需要另行确认后再写入规则。
