# vault-rules.yaml 声明层 + pjx-common-auditor v2.6.0

## 变更背景

此前所有规则散落在 pjx-common-auditor、hxc-obsidian-vault、CLAUDE.md 三处，每次审计需扫描文件系统。建立声明层 YAML 后，程序可直接校验确定性规则（目录、命名、Frontmatter、标签词表），AI 聚焦归属判断和内容质量。

## 变更内容

### 新建：`99-设置/vault-rules.yaml`

分层架构的声明层，覆盖 8 个维度：

| 维度 | 内容 | 可程序校验 |
|------|------|-----------|
| directories | 顶层目录角色、是否必须、子结构约定 | ✓ |
| frontmatter | 必填字段、type 枚举、日期格式 | ✓ |
| naming | MOC/日记/项目目录命名模板、禁止字符 | ✓ |
| moc | 量化阈值（分组数、正文行数、链接数） | ✓ |
| links | Wikilink 格式、首次唯一、别名规则 | 部分 |
| tags | 完整词表（领域、类型、状态、工具、人员） | ✓ |
| cross_reference | 发现路径定义、孤儿阈值、SVG 配套 | 部分 |
| attachments | 大文件阈值、目标路径、Git 策略 | ✓ |
| note_quality | 行数阈值、禁止措辞 | 部分 |
| scale_thresholds | 规模敏感开关（200 / 500） | ✓ |

### 修改：pjx-common-auditor v2.5.0 → v2.6.0

- Rule Authority 新增声明层为第一优先级
- KV-1/4/6/7/9/10 各维度添加 `声明层源` 引用行
- 声明层与 skill 关系明确：声明层 = What，skill = How / How severe

### 修改：hxc-obsidian-vault

- 开头添加声明层引用

## 设计决策

1. **YAML 放 99-设置/**：系统配置目录，与模板/配置文件并列
2. **声明层 vs 过程层边界**：信息存在 → YAML；判断 → skill
3. **标签词表首次完整 YAML 化**：此前只在 hxc-obsidian-vault 内联 HTML table，不可机读
4. **skill 不删除已有表**：保留 ✓/✗ 示例和分级逻辑，YAML 变更时标记冲突
