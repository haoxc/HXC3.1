# note-crafter 路由缓存优化

## 变更摘要

**日期**：2026-05-01
**变更**：在 note-crafter（Hermes + Codex 双版）执行策略中嵌入「路由速查」表，覆盖 80% 场景的关键词 → 目录映射。命中时窗口 2 无需读取 `vault-index.yaml`。

## 路由速查覆盖

从 vault-index.yaml 的 75 个关键词中提取高频路径，按主题分组：

| 主题 | 覆盖范围 |
|------|---------|
| Mac 工具（hermes/liteLLM/codex/alfred/shell） | `09-工具/mac-工具/` |
| 通用工具（obsidian/chrome/git/excel） | `09-工具/工具-通用工具汇编/` |
| AI/ML 术语（LLM/Agent/RAG/transformer） | `03-领域/31-人工智能学科/` |
| CS 基础（算法/数据结构/OS/编程语言） | `03-领域/30-计算机科学(CS)/` |
| 认知科学 | `03-领域/00-认知科学/` |
| 个人/商业/治理/资源 | 各自顶层目录 |

未命中 → 仍加载 vault-index.yaml 完整路由。
