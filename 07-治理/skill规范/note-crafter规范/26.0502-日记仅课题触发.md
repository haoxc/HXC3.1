# 日记追加条件化

**变更日期**：2026-05-02
**变更类型**：流程优化

## 变更内容

| 位置 | 变更 |
|------|------|
| Step 6 描述 | 「每篇笔记创建后」→「仅当笔记位于 `---/-workspaces/` 下时」 |
| Step 6 代码 | 新增前置 guard：`note_vault_path.startswith("---/-workspaces/")` 检查，不满足则 `print + exit` |
| 窗口 4 描述 | 追加「（仅课题类笔记）」 |
| Quality Checklist | 追加条件判断「若为课题类…否则跳过」 |
| 核心流程摘要 | 追加「→ 日记追加（仅课题类笔记）」 |

## 理由

用户反馈：每次创建笔记都触发日记更新，导致 `## 新建` 节被工具/领域笔记填满，失去课题追踪价值。工具笔记（如 hermes 配置踩坑）和领域笔记（如 Transformer 架构）不需要日记追溯；只有课题 workspace 下的笔记（课题产出、方案等）值得在日记中留痕。

## 判断逻辑

```
笔记 vault 相对路径
  ├── 以 ---/-workspaces/ 开头 → 触发日记追加
  └── 其他 → 跳过（print 日志，不报错）
```
