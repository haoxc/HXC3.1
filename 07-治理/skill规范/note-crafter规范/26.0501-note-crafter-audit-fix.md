# 26.0501-note-crafter-audit-fix

## 审计发现

- **P1** Step 2b/c `dirs[path]` 对子目录路径必然 KeyError——`idx["directories"]` 只有顶层 key，`search_routing` 返回的是子目录路径。核心优化路径不可用。
- **P2** search_routing 无匹配时无 fallback——新领域术语无法路由。
- **P2** 归属判断速查表缺 `05-资源/`。

## 修复

### P1: flatten 递归遍历 children 树

Step 2b 改写：新增 `flatten()` 递归函数，将嵌套 `directories` 树扁平化为 `{path: {moc, note_count, role, ...}}` 映射。容器 dict（无 path 字段）遍历 values。验证后 234 条路径全部可解析，moc 正确。

关键变更：
- 旧：`dirs = idx["directories"]` → `dirs[path]`（KeyError）
- 新：`flat = dict(flatten(idx["directories"]))` → `flat[path]`（正确）

### P2: fallback + 归属表补全

- 2b 末尾新增：`if not candidates: print("search_routing 无匹配，使用 Step 2e...")`
- 2e 表追加：`| 资源清单、书单 | 05-资源/ | 05-资源/05-资源.md |`
