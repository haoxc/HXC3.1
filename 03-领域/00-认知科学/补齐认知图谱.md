# 补齐认知图谱

**一句话定义**：补齐认知图谱是通过构建和维护结构化的概念关系网络，系统性填补认知缺口的过程——将隐性的「理解断层」转化为显性的「待连接节点」，再通过定向检索和推理将其补全。

## 问题：认知缺口从哪来

| 缺口类型 | 表现 | 示例 |
|----------|------|------|
| 概念缺口 | 听到术语但不理解其定义 | 讨论中提到「向量检索」但不知道它和关键词检索的区别 |
| 关系缺口 | 知道两个概念但不知道它们的关系 | 知道 RAG 和向量数据库，但不清楚 RAG 为什么需要向量数据库 |
| 因果缺口 | 知道现象但不知道原因 | 知道 LLM 有时「胡说」，但不知道为什么会有幻觉 |
| 边界缺口 | 知道核心但不知道适用范围 | 知道微调能提升效果，但不知道什么场景该微调 vs RAG |

## 核心方法：三阶段补齐循环

```
不知道自己不知道 → 发现缺口 → 构建图谱节点 → 定向补齐 → 知道自己知道
```

### 阶段一：发现缺口（Signal Detection）

| 信号 | 来源 | 动作 |
|------|------|------|
| 听到陌生术语 | 对话、阅读 | 记录为「待定义节点」 |
| 两个概念被并列但你不懂区别 | 对比陈述 | 记录为「待连接节点对」 |
| 某个判断你不理解依据 | 技术决策 | 记录为「待追溯因果边」 |
| 被问到答不上来 | 提问、自测 | 记录为「知识边界标记」 |

### 阶段二：构建图谱节点

每个认知缺口转化为一个图谱节点，包含：

| 属性 | 说明 | 示例 |
|------|------|------|
| 名称 | 概念名 | `向量检索` |
| 状态 | `未知` / `模糊` / `已理解` | `模糊` |
| 已有连接 | 已知的上下游关系 | `← 属于信息检索` / `→ 用于 RAG` |
| 缺口标记 | 具体缺失什么 | `不知与关键词检索的差异` |

### 阶段三：定向补齐

针对缺口类型选择补齐策略：

| 缺口类型 | 补齐策略 | 工具 |
|----------|---------|------|
| 概念缺口 | 查定义 → 对比相邻概念 → 写一句话定义 | 文档、论文、百科 |
| 关系缺口 | 画关系图 → 找最短路径 → 补连接边 | [[知识体系]]、知识图谱工具 |
| 因果缺口 | 追溯推理链 → 验证前提 → 补因果边 | 论文、源码、实验 |
| 边界缺口 | 找反例 → 标适用条件 → 补约束 | 案例研究、对比测试 |

## 图谱的两种形态

| 形态 | 载体 | 适用场景 |
|------|------|---------|
| 个人认知图谱 | 笔记双向链接（如 Obsidian [[Wikilink]]）、MOC 导航页 | 个人知识管理、学习 |
| Agent 认知图谱 | 结构化知识图谱（如 [[gbrain]] PGLite 图数据库） | AI Agent 跨会话记忆、三层混合搜索 |

> 两种形态共享同一套补齐方法论，区别在于图谱的持久化方式和查询能力。个人图谱靠手动维护链接，Agent 图谱靠自动实体提取和关系连线。

## AI Agent 场景的特殊性

AI Agent 的认知缺口比人类更结构化：

- **已知未知**：Agent 知道该查什么但没查到（如 API 文档缺失）
- **未知未知**：Agent 不知道自己缺少什么知识（跨会话遗忘）
- **工具**：[[gbrain]] 的三层混合搜索（关键词+向量+图谱）将 P@5 从 17.7%（无图谱）提升到 49.1%，核心就是自动补齐关系边
