## 一句话定义

口径是沟通双方对同一术语或概念的含义、边界、量纲达成的显式共识。

## 最小理解图

| 状态                | 结果          |     |
| ----------------- | ----------- | --- |
| 你说 A = 我听成 A'     | 口径不对齐，讨论无效  |     |
| 你说 A(宽) = 我说 A(窄) | 口径不一致，结论不可靠 |     |
| 双方对 A 的边界和量纲达成一致  | 口径对齐，讨论有效   |     |

## 判断标准

- 双方能否用同一句话定义该术语？
- 双方对该术语的边界是否有共识（什么算、什么不算）？

两项都是「是」→ 口径对齐。

## 容易混淆

| 对比 | 区别 |
|------|------|
| 口径 vs 定义 | 定义是单方面的术语说明；口径是双方对定义的**共识状态** |
| 口径 vs 术语统一 | 术语统一是规范层（规定所有人用同一个词）；口径对齐是沟通层（确认当下双方理解一致） |

## 典型例子

- 前端说「接口」指 HTTP API，后端说「接口」指 Java Interface → 口径不对齐
- PM 说「模块」指功能模块，开发说「模块」指代码模块 → 需要先对齐口径再讨论
- 「这个项目下周一交付」—— 周一上班时还是周一结束前？ → 时间量纲未对齐

## 压缩记忆

> 口径不对齐 = 鸡同鸭讲，双方以为在说同一件事，实际在说不同的事。
> 对齐口径 = 讨论开始前，先确认「你说的 X 具体指什么」。
