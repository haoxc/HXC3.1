# 术语辨析：层面 vs [[维度(Dimension)]] vs 角度
| 术语  | 核心定义                 | 核心侧重点           | 典型示例                           |
| --- | -------------------- | --------------- | ------------------------------ |
| 层面  | 事物的层级/深度（纵向/层级化划分）   | 强调“深浅、内外、上下”关系  | 问题的表面层面→本质层面；技术的应用层面→底层层面      |
| 维度  | 描述事物的独立衡量标准（横向/并列划分） | 强调“全面性、量化性、独立性” | 产品分析的功能维度、价格维度、体验维度；数据的时间/空间维度 |
| 角度  | 观察/分析的立场/视角（主观选择）    | 强调“立场、切入点、主观视角” | 从用户角度看需求；从管理者角度分析效率；从技术角度评估可行性 |
|     |                      |                 |                                |

## 核心区别要点
1. 层面：侧重「`纵向层级`」（如深浅、内外），有先后/包含关系；
2. 维度：侧重「`横向并列`」（如不同衡量维度），无主次、相互独立；
3. [[角度]]：侧重「`主观视角`」（如立场、切入点），随观察者选择而变化。
