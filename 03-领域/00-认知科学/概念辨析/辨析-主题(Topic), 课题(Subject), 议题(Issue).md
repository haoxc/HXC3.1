## 内容

### 主题 vs  课题(Subject)

| 维度     | 主题 (Topic)     | 课题 (Subject/Project)      |
| ------ | -------------- | ------------------------- |
| **范畴** | 宏观、宽泛、开放       | 微观、具体、闭环                  |
| **状态** | 静态的（一个领域）      | 动态的（一个过程或任务）              |
| **产出** | 综述、概览、讨论       | 结论、方案、成果                  |
| **类比** | “我想写关于‘森林’的东西” | “我想研究‘森林中某种真菌对松树根系的共生影响’” |

4. 结合之前的术语体系（定位参考）

如果将“主题”与“课题”放入您之前询问的 **Approach - Method - Solution** 体系中，它们的层级如下：

1. **主题 (Topic)**：这是最顶层的背景（Context）。
2. **课题 (Subject/Project)**：这是你选定的“战场”。
3. **Approach ([[方法(Approach)|进路]])**：你打算从哪个视角去切入这个课题。
4. **Method (方法)**：你解决这个课题的具体步骤。
5. **Technique (技术)**：你使用的微观手段。
6. **Solution (解决方案)**：课题最终交付的结果。

### 3. 主题(Subject)  vs 议题(Issue)

| **维度**  | **Subject (主题)** | **Issue (议题/问题)** |
| ------- | ---------------- | ----------------- |
| **性质**  | 中性的、描述性的         | 争议性的、挑战性的         |
| **目的**  | 告知、定义、分类         | 辩论、解决、做决定         |
| **稳定性** | 相对稳定（如：历史、数学）    | 具有时效性（如：目前的危机）    |
| **关联性** | 它是“容器”           |                   |
- [[议题(Issue)]]: **冲突（Conflict）与决策。** 它指代一个**有争议**、**需要关注**或**待解决**的具体点。如果一个 Subject 里出现了分歧，那个分歧点就是 Issue。
- 
## 2025 年的应用建议
在当下的科研或工作中，**“主题”向“课题”的转化**是成功的关键：
- **不要只选主题**：如果你只说“我的研究主题是 Svelte”，这太泛了。
- **要精准锁定课题**：你应该说“我的课题是 **Svelte 5 编译策略对 Web 端首屏加载速度的影响分析**”。
