## 内容
**FINER 原则**是学术研究和项目立项中用于评价“研究问题”或“课题”质量的经典标准。
**FINER 原则**是学术研究和项目立项中用于评价“[[研究问题]]”或“[[主题(Subject)]]”质量的经典标准.它能帮助你判断一个**课题（Subject/Project）**是否具有执行价值。
1. F - Feasible (可行性) —— “能不能做？”
	这是课题成功的基石。你需要评估资源是否匹配目标。
	
	- **样本与数据**：是否有足够的实验对象、受访者或数据集？（例如：AI 课题中是否有足够的标注数据 [1]）。
	- **技术与工具**：当前的技术水平（如 Svelte 5 的稳定程度）能否支撑开发？
	- **时间与经费**：是否能在规定期限内完成？预算是否超标？
2. I - [[Interesting (趣味性/吸引力)]] —— “想不想做？”
	指课题对研究者本人及其同行、导师或投资人的吸引力。
	- **个人驱动**：你是否对该主题有长期的热情？
	- **同行关注**：该问题是否是当前领域的热点（如 2025 年关注的“生成式 AI 的端侧部署”）？

3. N - [[Novel (新颖性)]] —— “是不是首创？”
	研究必须能为现有知识体系提供“增量”，而不是简单的重复。
	- **新发现**：提供了前人未曾发现的数据或规律。
	- **新视角**：用不同的 **Approach** 重新审视旧问题（例如：用 Svelte 的编译视角去优化 React 已有的性能痛点）。
	- **改进**：对现有 **Method** 或 **Solution** 的显著优化。
4. E - [[Ethical (伦理合规)]] —— “准不准做？”
	在 2025 年，伦理审查变得愈发严格，特别是涉及 AI 和隐私的课题。
	- **数据隐私**：是否符合 GDPR 或国内的《数据安全法》？
	- **实验对象**：是否保护了人类受访者的隐私和心理健康？
	- **AI 伦理**：算法是否存在偏见或歧视？
5. R - Relevant (相关性/重要性) —— “值不值得做？”
	课题必须对科学进步、政策制定或行业实践有实际意义。
	- **学术价值**：能否填补某个理论空白？
	- **应用价值**：能否解决现实中的具体问题（如：提高 10% 的系统吞吐量）？
## 总结：如何应用 FINER 原则优化你的课题？
你可以通过下表对你的“课题”进行自测打分（1-5分）：

| 维度              | 自测问题                                  | 得分  |
| --------------- | ------------------------------------- | --- |
| **Feasible**    | 我手里有做这件事所需的硬件、软件和时间吗？                 |     |
| **Interesting** | 我在向别人介绍这个想法时，他们会感到兴奋吗？                |     |
| **Novel**       | Google Scholar 或 GitHub 上能搜到一模一样的东西吗？ |     |
| **Ethical**     | 这个研究会侵犯他人隐私或造成安全风险吗？                  |     |
| **Relevant**    | 做完之后，对我的行业或学术履历有实质帮助吗？                |     |
|                 |                                       |     |
