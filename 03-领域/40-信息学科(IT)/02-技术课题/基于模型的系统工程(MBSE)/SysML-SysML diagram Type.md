### 一、SysML核心组成：9类图表+1类分配表
SysML由9种不同功能的图表类型，以及用于在不同图表类型间映射语言元素的分配表（Allocation Table）构成，具体可分为**结构图表（Structure Diagrams）** 和**行为图表（Behavior Diagrams）** 两大类，各类别包含的图表如下：
- **结构图表**：侧重系统静态结构的定义与展示，包括需求图（Requirement diagram，req）、块定义图（Block Definition Diagram，bdd）、内部块图（Internal Block Diagram，ibd）、参数图（Parametric Diagram，par）、包图（Package diagram，pkg）。
- **行为图表**：侧重系统动态行为的描述，包括活动图（Activity diagram，act）、序列图（Sequence diagram，sd）、状态机图（State Machine diagram，stm）、用例图（Use Case diagram，uc）。
- **分配表（Allocation Table）**：本质是关系矩阵，无特定图表类型属性，用于跨图表类型映射语言元素。

### 二、SysML图表分类属性详情（基于表格）
通过多维度属性对比，可清晰区分各类图表的差异与适用场景，核心属性维度及关键信息如下：

| 图表名称      | 图表类型       | UML 2对应类比   | 软件开发生命周期（SDLC）用途 | 核心特性                                   |
| --------- | ---------- | ----------- | ---------------- | -------------------------------------- |
| 需求图（req）  | 静态结构[声明式]  | N/A（无对应）    | 需求分析             | 无动态/数学仿真能力，仅用于明确系统需求                   |
| 用例图（uc）   | 行为[不可仿真]   | 用例图         | 需求分析             | 虽被归为行为图表，但行为语义模糊不完整，**非图灵完备**，无法仿真     |
| 活动图（act）  | 动态行为[可仿真]  | 活动图[轻微修改]   | 系统分析、功能分析、系统设计   | **图灵完备**，支持动态仿真（模拟系统时变行为），可用于梳理系统功能流程  |
| 序列图（sd）   | 动态行为[可仿真]  | 序列图         | 系统设计             | 图灵完备，支持动态仿真，侧重展示对象间交互的时间顺序             |
| 状态机图（stm） | 动态行为[可仿真]  | 状态机图        | 系统分析、系统设计        | 图灵完备，支持动态仿真，用于描述对象在不同状态下的转换逻辑          |
| 块定义图（bdd） | 静态结构[黑盒定义] | 类图[中度修改]    | 系统分析、系统设计        | 定义系统中“块”（核心构建单元）的外部属性与关系，不展示内部结构       |
| 内部块图（ibd） | 静态结构[白盒使用] | 复合结构图[中度修改] | 系统分析、系统设计        | 展示“块”的内部组成、连接关系，呈现系统内部结构细节             |
| 参数图（par）  | 静态结构[白盒使用] | N/A（无对应）    | 系统分析、系统设计        | 唯一支持**数学建模与仿真**的SysML图表，通过数学方程定义系统行为模型 |
| 包图（pkg）   | 静态结构[分组]   | 包图[轻微修改]    | 所有SDLC阶段         | 用于对系统模型元素（如图表、块、需求等）进行分组管理，提升模型组织性     |
| 分配表       | N/A[关系矩阵]  | N/A（无对应）    | 所有SDLC阶段         | 无仿真、代码生成等能力，仅用于跨图表映射元素关系（如需求与功能的对应）    |

### 三、关键术语与补充说明
1. **动态仿真（Dynamic Simulation）**：指计算机程序模拟系统时变行为的能力，除用例图外，其余SysML行为图表（活动图、序列图、状态机图）均具备此能力。
2. **数学建模与仿真（Mathematical ModSim）**：通过数学方程定义系统模型并执行仿真，仅参数图（par）支持该功能，其他SysML/UML 2图表均不具备。
3. **图灵完备（Turing Complete）**：指图表的行为描述能力可实现任意可计算逻辑，活动图、序列图、状态机图满足此条件，用例图不满足，因此无法仿真。
4. **黑盒/白盒**：“黑盒”（如块定义图）仅关注外部属性与接口，不涉内部；“白盒”（如内部块图、参数图）需展示内部结构或数学逻辑。
