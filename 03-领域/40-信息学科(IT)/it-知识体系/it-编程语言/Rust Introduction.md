## 概述

Rust 是一门强调可靠性、性能与工程生产力的系统编程语言。它的核心目标不是让语法更短，而是在没有垃圾回收器的前提下，通过类型系统、所有权模型和编译期检查，尽量把内存安全、并发安全等问题提前暴露。

一句话理解：Rust 试图把 C/C++ 级别的性能控制，与现代语言的安全边界和工具链体验结合起来。

## 核心定位

| 维度   | Rust 的定位                             |
| ---- | ------------------------------------ |
| 语言类型 | 编译型、静态类型、系统编程语言                      |
| 核心目标 | 高性能、内存安全、并发安全、可维护                    |
| 典型对标 | C、C++、Go、部分场景下的 TypeScript/Python 后端 |
| 主要代价 | 学习曲线较陡，尤其是所有权、借用、生命周期                |
| 工具链  | `rustup` 管理工具链，`cargo` 管理项目、依赖、构建与测试 |

## 最小理解图

```text
性能敏感的软件需求
  ↓
静态类型 + 编译期检查
  ↓
所有权 / 借用 / 生命周期
  ↓
无 GC 的内存安全与并发安全
  ↓
适合系统、基础设施、CLI、WebAssembly、嵌入式等场景
```

## 为什么 Rust 特别

### 1. 所有权模型

Rust 中每个值都有一个所有者。值离开作用域时会被自动释放。这个机制让 Rust 不依赖垃圾回收器，也不需要程序员手动 `free`。

关键规则可以压缩成三句话：
1. 每个值同一时间只有一个所有者。
2. 所有者离开作用域，值会被释放。
3. 可以通过借用访问值，但借用必须满足编译器规则。

### 2. 借用与可变性约束

Rust 允许引用一个值，但会限制可变引用和不可变引用的同时存在。粗略理解：
- 可以有多个不可变借用。
- 同一时间只能有一个可变借用。
- 借用不能比被借用的数据活得更久。

这套规则看起来严格，但目的很明确：避免悬垂指针、数据竞争和难以定位的内存错误。

### 3. Cargo 工具链

Cargo 是 Rust 的项目管理、依赖管理、构建、测试和发布工具。入门 Rust 时，不应只学 `rustc`，而要直接用 Cargo 理解 Rust 的工程工作流。

常用命令：

```bash
cargo new hello-rust
cd hello-rust
cargo run
cargo test
cargo build --release
```

## 适用场景

| 场景          | 为什么适合 Rust                            |
| ----------- | ------------------------------------- |
| CLI 工具      | 单文件部署、性能好、依赖清晰，适合替代部分 Shell/Python 工具 |
| 系统软件        | 可控制内存与性能，同时降低 C/C++ 常见内存错误            |
| 网络服务        | 性能强，适合高并发、低延迟场景                       |
| WebAssembly | Rust 到 WASM 的生态成熟，适合浏览器/边缘计算中的性能模块    |
| 嵌入式         | 无 GC、可控资源占用，适合资源受限设备                  |
| 基础设施        | 适合构建编译器、数据库、运行时、开发工具等底层组件             |

## 不适合的场景

- 只是写一次性脚本，且性能与可靠性要求不高。
- 团队没有时间理解所有权、借用和生命周期。
- 业务变化极快，`原型速度`比长期可靠性更重要。
- 已有生态主要在 Python/JavaScript，Rust 只会增加协作成本。

## 和相近语言的区别

| 对象         | 差异                                                |
| ---------- | ------------------------------------------------- |
| C/C++      | Rust 保留底层控制能力，但通过编译期规则减少内存错误；代价是语言规则更严格。          |
| Go         | Go 更重视简单、快速开发和并发工程体验；Rust 更重视性能控制和无 GC 场景。        |
| Python     | Python 适合快速表达和数据/自动化生态；Rust 适合把性能敏感、稳定性要求高的模块工程化。 |
| TypeScript | TypeScript 强在 Web/应用层生态；Rust 更接近系统层、性能层和基础设施层。    |

## 入门路径

1. 先安装 `rustup` 和 `cargo`，建立工具链。
2. 用 `cargo new` 创建第一个项目，不从裸 `rustc` 开始。
3. 先理解变量、函数、结构体、枚举、模式匹配。
4. 再集中突破所有权、借用、生命周期。
5. 写一个小 CLI 工具，练习参数解析、文件读写、错误处理。
6. 学会读编译器错误信息，把它当成“辅导反馈”，不是单纯报错。

## 第一批要掌握的关键词

| 关键词 | 最小解释 |
| --- | --- |
| `rustup` | Rust 工具链安装与版本管理工具 |
| `cargo` | Rust 项目、依赖、构建、测试管理工具 |
| ownership | 所有权，决定值由谁负责释放 |
| borrowing | 借用，在不转移所有权的情况下访问值 |
| lifetime | 生命周期，描述引用有效范围 |
| crate | Rust 的包/编译单元概念 |
| trait | 类似“能力接口”的抽象机制 |
| enum | 枚举，Rust 中非常核心的数据建模方式 |
| pattern matching | 模式匹配，常与 `match` 一起使用 |

## 典型学习误区

- 只把 Rust 当成“更难的 C++”。Rust 的难点不在语法堆叠，而在所有权带来的思维方式变化。
- 一开始就追求写复杂泛型和宏。初学阶段更应关注数据建模、错误处理和所有权。
- 把编译器报错视为阻碍。Rust 编译器的错误提示往往就是学习路径的一部分。
- 用 Python/JavaScript 的动态写法硬套 Rust。Rust 更鼓励先明确数据结构和所有权关系。
