# 六力模型 (Six Forces Model)

### 1. 🎯 一句话定义 (Definition)

> **六力模型是一个评估组织或平台“[[竞争环境]]”与“[[生存能力]]”的战略工具，通过分析六种影响核心竞争力的外部力量，来判断其长期的盈利能力与发展潜力。**

### 2. ⚡ 核心特征 (Key Characteristics)

- **系统性**：它不再只关注内部，而是将视角拉升到整个市场生态。
- **动态性**：六种力量的强弱会随着技术突破（如 AI 的出现）和政策变化而实时波动。
- **结构化**：它将复杂的市场环境拆解为六个标准的模块，便于进行标准化的对比分析。
- **博弈性**：每一力都代表了一种利益冲突，模型的核心在于寻找平衡点或突破点。

### 3. 🚀 应用场景 (Use Cases)

- **软件平台评估**：评估一个平台在生态中的地位（如：它对开发者的议价能力，它的替代成本）。
- **企业战略规划**：判断进入一个新领域的风险。
- **行业分析报告**：分析为什么某个行业（如云计算）目前处于高门槛、高竞争的状态。

### 4. 📍 战略定位 (Positioning)

- **经典六力组成**：
    
    1. **现有竞争者的竞争 (Existing Rivalry)**：同类平台之间的“内卷”程度。
    2. **潜在进入者的威胁 (New Entrants)**：行业门槛高吗？新玩家容易进来搅局吗？
    3. **替代品的威胁 (Substitutes)**：是否有颠覆性的新技术（如低代码平台替代传统开发平台）？
    4. **供应商的议价能力 (Supplier Power)**：底层资源（如芯片、云服务器）是否掌握在少数人手里？
    5. **购买者（客户）的议价能力 (Buyer Power)**：客户的选择多吗？他们是否有强大的砍价权？
    6. **互补品/生态力 (Complementary Products)**：这是第六力，指与平台配套的生态丰富度（如 iPhone 与 App Store 的关系）。

### 5. 💡 教练私房话 (Coach's Tip)

- **沟通贴士 (降维打击)**：
    - 当别人在讨论产品好不好用时，你直接谈**“转换成本”**（隶属于替代品/购买者力量）。
    - 话术：“虽然方案 B 的 UI 更好看，但从**六力模型**来看，方案 A 拥有更强的**生态力 (Complementary)**，我们的其他系统可以无缝集成，这能极大降低我们的整体**转换成本**。”
- **实战建议**：在进行软件平台评估时，特别关注**“第六力”**。在现代软件竞争中，技术差距在缩小，**生态的厚度**往往决定了平台的生杀大权。
