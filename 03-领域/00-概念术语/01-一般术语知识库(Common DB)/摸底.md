## 一句话定义

摸底是在正式决策、规划或行动前，对对象的当前状态、底数、差距和风险做一次低成本的初步掌握。

## 最小理解图

```text
未知或不清楚的对象
  ↓
快速收集关键事实
  ↓
形成底数、边界、风险判断
  ↓
决定下一步是否深入调研、立项或执行
```

## 判断标准

1. 目标是先“知道大概情况”，不是一次性得出最终结论。
2. 关注底数、结构、差距、风险和约束，而不是只收集零散材料。
3. 输出应能支撑下一步判断：继续深入、暂停、转向、立项或制定方案。

## 和相近概念的区别

| 容易混淆对象 | 区别 |
| --- | --- |
| 调研 | 调研通常更系统，可能有明确方法、样本、访谈和分析框架；摸底更偏前置、快速、粗颗粒。 |
| 盘点 | 盘点偏“有什么、多少、在哪”；摸底还要判断现状背后的差距、风险和下一步动作。 |
| 评估 | 评估通常要给出价值、质量或达标判断；摸底主要解决“情况是否清楚”。 |
| 验收 | 验收是事后确认成果是否符合标准；摸底是事前了解基础条件。 |
| 基线 | 基线是经确认后可作为比较参照的状态；摸底可能为形成基线提供原始输入。 |

## 典型例子

- 项目启动前，先摸底现有系统、数据、人员、流程和历史问题。
- 课程改革前，先摸底学生基础、教师资源、设备条件和企业需求。
- 工具迁移前，先摸底现有配置、依赖、使用习惯和风险点。

## 常见误区

- 把摸底做成漫无边际的信息收集，最后没有可判断的底数。
- 过早追求完整性，把本该快速完成的前置动作拖成大型调研。
- 只问“有什么”，不问“缺什么、卡在哪里、下一步怎么判断”。

## 落地模板

| 项目 | 要问的问题 |
| --- | --- |
| 对象 | 这次要摸清谁或什么？边界到哪里为止？ |
| 底数 | 已有多少、分布如何、当前状态如何？ |
| 差距 | 和目标、标准或预期相比差在哪里？ |
| 风险 | 哪些不确定因素会影响后续决策？ |
| 下一步 | 需要深入调研、直接执行、暂缓，还是调整目标？ |

## 压缩记忆

> 摸底不是把问题彻底研究完，而是先把“现在到底是什么情况”看清到足以决定下一步。
