# 色谱理论与视觉编码 (Color Palette Theory)

> [!important] 核心原则
> 色谱的选择必须由**数据类型**决定。根据数据的逻辑关系（定性、定量、发散），我们需要匹配不同的色彩映射策略。

> [Color for data visualization - Spectrum](https://spectrum.adobe.com/page/color-for-data-visualization/)
---

## 1. 三大核心色谱类型

### A. 顺序型色谱 (Sequential)
* **适用数据**：有明显先后顺序、连续的数值（如：温度、降雨量、销售额）。
* **逻辑**：利用**亮度 (Lightness)** 的阶梯式变化。通常从浅色（低值）到深色（高值）。
* **沟通贴士**：人眼对亮度的敏感度远高于色相的变化，因此单色系的深浅是***最不容易产生歧义的***。
### B. [[发散型色谱 (Diverging)]]
* **适用数据**：有一个关键的`中性中心点`（如：利润的正负、气温偏差、支持率）。
* **逻辑**：两个互补色向中间的白色或灰色汇聚。
* **沟通贴士**：中心点的颜色必须是中性的。在汇报盈亏时，这是一种极具**情绪导向**的视觉工具。
### C. 定性型色谱 (Qualitative / Categorical)
* **适用数据**：互不隶属的类别（如：产品分类、不同省份、水果种类）。
* **逻辑**：利用不同的**色相 (Hue)**，且各颜色在亮度上应尽量接近，以示地位平等。
* **沟通贴士**：颜色数量不宜超过 7-10 个，否则大脑的检索压力会激增。

## 2. 视觉检索中的“陷阱”：彩虹色谱 (Jet / Rainbow)

> [!warning] 警告
> 在学术和专业可视化界，**彩虹色谱（Rainbow Map）通常是被禁止的**。
* **原因**：彩虹色的亮度变化是不均匀的（例如黄色看起来比蓝色亮得多），这会导致用户在检索信息时看到并不存在的“伪边界”。
* **改进方案**：改用 **[[viridis色谱]]** 或 **[[Magma色谱]]** 等“感知均匀色谱”。这些色谱即使在黑白打印下，其物理亮度依然是线性的。

## 3. 沟通高手实战：如何用语言描述色谱选择？
当你在向设计或技术团队下达指令时，使用以下专业表述能瞬间提升你的“段位”：
* **强调对比度**： “我们需要增加这个色谱的**感知对比度 (Perceptual Contrast)**，确保弱视或色盲用户也能分辨出核心趋势。”
* **解释色谱逻辑**： “因为我们的数据是关于‘同比增长率’的，带有一个零点值，所以我建议采用**发散型色谱**，用红色代表负值，蓝色代表正值。”
* **视觉一致性**： “为了保持**语义关联性**，我们应将‘自然灾害’类别映射为暖色调，‘安全区域’映射为冷色调。”

## 4. 色彩与心理索引 (Color Indexing)

在视觉检索中，颜色具有**直觉索引**功能：
* **红色**：停止、危险、亏损、高热。
* **蓝色**：平稳、可靠、寒冷。
* **绿色**：通过、增长、环保。
