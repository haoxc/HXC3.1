### ReAct 与其他决策框架的对比

| 框架       | 核心优势       | 适用场景                       | 与 ReAct 关系                    |
| -------- | ---------- | -------------------------- | ----------------------------- |
| ReAct    | 迭代行动、工具交互强 | 复杂多步骤 + 工具调用任务（如故障排查、数据分析） | 主打 “行动落地”，适合需工具协作的场景          |
| CoT（思维链） | 纯思考、无工具交互  | 逻辑推理任务（如数学题、代码调试）          | ReAct 的子集（ReAct = CoT + 工具行动） |
| ToT（思维树） | 多分支探索、全局规划 | 超高复杂度任务（如科研论文撰写、复杂决策）      | 比 ReAct 更灵活，但实现成本高            |

**选型建议**：大多数业务场景（如智能客服、数据分析、自动化运维）用 ReAct 足够，结合 Function Calling 即
