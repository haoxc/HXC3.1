## 概述
`browser-use` 是一个专门为 LLM 设计的 Python 库，旨在让 AI 能够像人类一样操作浏览器。

- **核心理念**：它不仅是自动化工具（如 Playwright），它还自带了**多模态感知**能力。
- **核心组件**：
    - **感知 (Perception)**：将复杂的 HTML DOM 树提取为 AI 易于理解的“可交互元素列表”，并结合截图提供视觉上下文。
    - **行动 (Action)**：支持点击、输入、滚动、多标签切换、甚至执行自定义 Python 脚本。
    - **大脑协同**：原生支持 LangChain，可以无缝对接 GPT-4o、Claude 3.5 或 Gemini。
## 2. Browser-use vs. Chrome MCP：深度对比

这是你在做架构选择时最需要理清的区别：

| **特性**   | **Browser-use (Python 框架)**   | **Chrome MCP (协议/工具)**               |
| -------- | ----------------------------- | ------------------------------------ |
| **形态**   | 一个 Python 库，用于构建独立的 Agent 应用。 | 一个协议，用于让现有 Agent（如 Claude）连接 Chrome。 |
| **运行环境** | 开发者在服务器或本地运行 Python 脚本。       | 在 MCP 客户端（如 Claude Desktop）中直接调用。    |
| **控制深度** | 强于“网页交互”，模拟用户行为。              | 强于“调试/性能”，可以直接查看网络包和控制台。             |
| **开发成本** | 需要编写 Python 代码。               | 几乎零代码，只需配置 `config.json`。            |
| **适用对象** | 想要构建 **自建平台/产品** 的开发者。        | 想要提升 **个人生产力/辅助编程** 的用户。             |
