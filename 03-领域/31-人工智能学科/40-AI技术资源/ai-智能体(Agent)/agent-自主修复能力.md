## 内容
在 Agent 自主修复能力（Self-Healing / Auto-Fixing） 维度，这反映了` AI 发现错误、分析日志并重新规划路径`的闭环能力。

## 1. 核心选手评测

## Claude Code

- 修复逻辑：Agentic Loop（原生代理循环）。
- 表现：它是目前对 Claude 3.5 Sonnet 的 `tool_use`（工具调用）控制得最精细的。当它执行 `npm test` 失败时，它不会立刻问你，而是自动执行 `cat` 读取报错日志，在内存中修正代码逻辑，然后再次尝试运行，直到测试通过。
- 评分：⭐⭐⭐⭐⭐ (最稳健，逻辑一致性极高)

## Aider (Git 驱动协作派)

- 修复逻辑：Test-Driven Repair（测试驱动修复）。
- 表现：Aider 的自主性略低，它更倾向于协同。你需要告诉它 `/run npm test`，如果报错，它会把报错信息自动加入上下文。它的强项在于：一旦修复，它会自动生成极高质量的 Git Commit。它不会在后台反复“偷跑”你的命令，除非你明确指示。
- 评分：⭐⭐⭐⭐ (更安全、可控，适合人类审核)

## Roo Code / Cline (全自动执行派)

- 修复逻辑：Autonomous Task Execution（自主任务执行）。
- 表现：这是目前最“激进”的。你可以给它开启 "Always Allow" 模式，它会自己开终端、查进程、杀掉占用端口的程序、改代码、重启服务。它在处理“环境配置错误”（如数据库连不上、依赖缺失）时的修复能力极强。
- 评分：⭐⭐⭐⭐⭐ (自主性最强，但也最费 Token)

## OpenHands / Devin 类 (长程规划派)

- 修复逻辑：Evaluation Loop（评估循环）。
- 表现：它们会在沙盒里反复运行 `lint` 和 `unit tests`。如果修复失败，它们会尝试从历史记录中寻找“什么时候开始错的”，并进行回溯式修复。
- 评分：⭐⭐⭐⭐ (深度高，但响应速度慢)

## 2. 综合能力对比矩阵

| 特性       | Claude Code | Aider      | Roo Code | OpenHands |
| -------- | ----------- | ---------- | -------- | --------- |
| 报错捕获方式   | 自动实时监听      | 需指令触发      | 自动执行+截图  | 持续集成监控    |
| 修复策略     | 逻辑重推理       | 对话式补丁      | 环境+代码全改  | 全局重构修复    |
| Token 消耗 | 中等          | 低 (最省钱)    | 高        | 极高        |
| 适用场景     | 复杂业务逻辑修复    | 快速 Bug Fix | 系统环境配置   | 攻克陈年顽疾    |

## 💡 深度建议

基于你目前使用 OhMyGPT 的情况，针对“自主修复”的选型建议：
1. 如果你心疼钱 (Token)：选 Aider。它不会在后台反复尝试导致余额跳水，修复过程完全透明。
2. 如果你想当“甩手掌柜”：选 Roo Code 或 Claude Code。它们能解决那种“因为我少装了一个库导致程序跑不起来”的琐碎问题。
3. 针对 Postgres + Docker：推荐 Roo Code。因为它能感知容器状态，如果 Docker 没启动，它会规划出 `docker start` 的步骤，这种环境级修复是它的杀手锏。
