## GitHub 提交提醒

- 当前 `98-附件` 总量约 5GB，且存在多个超过 100MB 的文件，不能直接进入普通 GitHub 仓库。
- 当前策略是：保留本地附件，提交索引与知识文本；大文件如确需同步，应单独使用 Git LFS、网盘或对象存储。
- 后续新增超过 10MB 的附件，优先放入本目录对应主题子目录，不再散落在业务目录的 `z附件` 中。
