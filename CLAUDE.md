# CLAUDE.md

## 笔记风格

创建或编辑笔记时遵循。每组规则含反例对照。

### 标题

| 规则 | ✓ | ✗ |
|------|---|---|
| 见名知意，不加"关于"、"如何"等冗余词 | `PostgreSQL 索引优化` | `关于 PostgreSQL 索引的一些优化方法` |
| 名词为主，动作型用动名词 | `环境配置指南` | `如何配置环境` |

### 长度

| 规则 | 量化 |
|------|------|
| 核心结论在前 15 行内 | 读者不滚动就能看到结论 |
| 展开细节用 `## 细节` 二级标题收束 | 折叠后正文保持简洁 |
| 超过 80 行考虑拆成子页面 | 用 `[[子页面]]` 链接引出 |

### 句式

| 规则 | ✓ | ✗ |
|------|---|---|
| 陈述句，直接下结论 | `该方案覆盖三个场景。` | `我认为这个方案可能比较合适。` |
| 每个结论自带理由 | `选 Redis：内存操作延迟 < 1ms，比 PostgreSQL 快两个数量级。` | `选 Redis 比较好。` |
| 禁止口语化表达 | `性能显著优于方案 B。` | `方案 A 简直秒杀方案 B。` |
| 禁止情绪化词语（惊人、完美、超级） | `覆盖率达到 95%。` | `这个结果太惊人了！` |

### 结论段模板

技术决策或分析型笔记中，每个结论用一个三段式段落：

```
**结论**：一句话判断。
**理由**：为什么这个判断成立（1-3 条）。
**出处**：数据来源 / 实验条件 / 参考文档。
```

| ✓ | ✗ |
|---|----|
| `**结论**：选 Redis 做会话存储。\n**理由**：内存操作 < 1ms，比 PostgreSQL 快两个数量级；原生支持 TTL 过期。\n**出处**：Redis 7.0 官方 benchmark，2024-03。` | `我建议用 Redis，因为它很快，而且支持过期。` |

适用场景：方案选择、技术评估、性能分析、错误分析。日记和纯罗列型笔记不适用。

### 对比

> 能用表格不用段落。能用列表不用表格（当维度 ≤2 时）。

| 场景 | 推荐格式 | 理由 |
|------|----------|------|
| 两方案对比 | 三列表格（维度 \| A \| B） | 多列对齐扫描快 |
| 单维度罗列 | 无序列表 | 表格只有一列就是浪费 |
| 版本差异 | 二列表格（版本 \| 变更） | 两列刚好 |
| 配置参数 | 二列表格（参数 \| 说明） | 快速定位参数名 |
| 步骤流程 | 有序列表 | 顺序是核心信息，表格打断阅读流 |

### 链接

| 规则 | ✓ | ✗ |
|------|---|---|
| 首次出现时链接，同篇内不重复 | 开头 `[[向量检索]]` 一次 | 每个段落都写 `[[向量检索]]` |
| 提到就链（首次），不重复解释 | `详见 [[RAG 架构]]` | 在正文展开 RAG 概念 |
| 链接文本 = 目标笔记标题 | `[[Transformer 架构]]` | `[[Transformer 架构\|这里]]` |
| 别名仅用于必要场景 | `[[Python\|Python 语言]]`（消歧义） | `[[Python\|点我]]`（无意义重命名） |

> 别名 `[[真实标题\|显示文本]]` 仅用于：① 消歧义（同名不同义）；② 语法需要（复数/加词使句子通顺）。禁止用于缩短、美化或无意义重命名。

### 语气

| 规则 | ✓ | ✗ |
|------|---|---|
| 冷静精确，像给一年后的自己留纸条 | `PyTorch 2.0 起 torch.compile() 替代了大部分手动优化。` | `PyTorch 现在超级好用，强烈推荐升级！` |
| 中文为主，英文术语括号保留 | `智能体(Agent)通过规划(Planning)选择工具。` | `Agent 通过 planning 来选择 tools。` |
| 不加感叹号 | `注意：API 有速率限制。` | `注意！！！` |

### 结尾

| 规则 | ✓ | ✗ |
|------|---|---|
| 写完了就停，不凑数 | 最后一个事实写完就结束 | 加一段 "总之..." 或 "综上所述..." |
| 不要"总结"段（除非是专门的总结页） | 正文结束即止 | `## 总结\n\n通过以上分析可以看出...` |

## 文件命名

| 规则 | 示例 |
|------|------|
| 文件名 = 笔记标题（中文） | `PostgreSQL 索引优化.md` |
| MOC 用文件夹笔记 `{目录名}.md` | `03-领域/AI/AI.md` |
| 项目目录日期前缀 | `2026-04-29-项目名/` |
| 避免空格和特殊字符 | 用连字符 `-` 不用下划线 `_` |

## 底层逻辑

1. **检索速度 > 阅读体验** — 标题决定搜不搜得到，表格和清单比段落扫描快
2. **写给未来的陌生人** — 每个结论自带理由，不留隐式知识
3. **一棵树一张嘴** — 每个概念只在一处详细解释，其他地方用链接引用

- 锚点链接：`[[笔记标题#章节]]`
