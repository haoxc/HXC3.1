## 内容
**GeoPackage**（扩展名为 `.gpkg`）是一种开放的、基于标准的地理空间数据存储格式。它本质上是一个 **SQLite 数据库容器**，旨在通过单个文件统一存储和交换多种类型的地理信息。

### 核心目的
- **消除“文件碎片”**：取代传统 Shapefile 需要 3-8 个关联文件（`.shp`, `.dbf`, `.prj` 等）的麻烦，实现**单文件**管理。
- **全能存储**：在一个数据库中同时存储矢量要素（点/线/面）、栅格瓦片（影像）、属性表和元数据。
- **跨平台兼容**：作为 [OGC（开放地理空间联盟）](https://www.ogc.org/standards/geopackage/) 标准，它在 Windows、Linux、macOS 以及 Android/iOS 移动端具有极高的通用性。
- **高效性能**：内置 R-tree 空间索引，在大型数据集的加载、平移和空间查询上比传统格式更快。
### 主要优势

| 特性             | 说明                                                               |
| -------------- | ---------------------------------------------------------------- |
| **无容量限制**      | 突破了 Shapefile 单文件 2GB 的上限，理论支持高达 **140TB** 的数据量。                 |
| **支持 Unicode** | 完美支持中文字符，解决了 Shapefile 常见的属性表**乱码**问题。                           |
| **SQL 驱动**     | 因为它是 SQLite 数据库，你可以直接使用标准 **SQL 语句** 进行复杂的空间查询和分析。               |
| **样式保存**       | 可以直接在文件中保存图层渲染样式（如 [QGIS](https://qgis.org/) 样式），确保数据在不同软件间展现一致。 |
|                |                                                                  |

### 典型应用场景

1. **离线移动作业**：由于其轻量、单文件的特性，它是移动端 GIS（如 QField、ArcGIS Maps SDK）首选的离线地图包格式。
2. **数据交换与发布**：作为现代 GIS 数据交换的标准格式，正逐渐取代 Shapefile 成为科研和政府部门发布数据集的首选。
3. **复杂制图项目**：将一个地图项目所需的所有矢量图层和背景影像图块全部打包在一个 `.gpkg` 文件中，方便团队协作分享。

>  - **GeoPackage** 已成为主流软件（如 QGIS 3.x）的**默认存储格式**
