## 场景, 风格, 场景风格
简单来说，**“场合”是[[命题]]，“风格”是解题。** 掌握这个逻辑能让你在任何社交场景中表现得既得体又有自信。
### 含义对比：场景 vs. 状态

| 维度       | 场合 (Occasion / Context) | 场合风格 (Dress Code / Vibe)    |
| -------- | ----------------------- | --------------------------- |
| **定义**   | **时间、地点、人物、目的**的集合。     | 应对该场合所选取的**视觉语言**。          |
| **核心逻辑** | 回答“我在哪？和谁？干什么？”         | 回答“我该呈现出什么样的状态？”            |
| **约束力**  | 是客观存在的**边界**（外部要求）。     | 是主观表达的**选择**（个人诠释）。         |
| **举例**   | 互联网公司面试。                | 商务休闲（Smart Casual），显得专业且有活力 |

## 关系
### 派生关系
- 派生不是改变你的本质，而是调整风格的 **“音量”**。

## 风格- 场景风格
### 1. 概念层级：父项与子项
- **风格（Style）是“元实体” (Meta-entity)：** 它处于逻辑的高位，是**全局性、通用性**的描述符（如：简约、专业、前卫）。它不依赖于具体的外部条件，是核心特征的抽象。
- **场景风格（Scenario-based Style）是“实例” (Instance)：** 它是风格在特定约束条件下的**具体实现**。它是“风格”叠加了“时间、空间、人物”等维度后的派生结果。
### 2. 抽象程度对比（维度多寡）

- **风格更抽象（维度少，内涵广）：**
    它是**一维**的特征定义。在元数据模型中，它通常作为一个“枚举值”或“标签”。
    - _特征：_ 稳定性强、复用率高、描述粒度粗。
- **场景风格更具体（维度多，外延窄）：**
    它是**多维**的数据组合。它必须由 `风格 + 场景变量` 运算得出。
    - _公式：_  $Style_{Scene} = f(Style, Context)$
    - _特征：_ 动态性强、上下文相关、描述粒度细。
### 3. 数据映射逻辑

在元数据管理中，两者的关系体现了**派生逻辑**：
- **风格**是 **“[[主数据]]” (Master Data)** ：定义了“我是谁”的基本属性。
- **场景风格** 是 **“应用数据” (Application Data)**：定义了“我在特定环境下呈现出的样子”。

| 维度       | 风格 (Style)  | 场景风格 (Scenario Style) |
| -------- | ----------- | --------------------- |
| **抽象层级** | **高**（元模型层） | **低**（应用层）            |
| **存在形式** | 静态的**核心属性** | 动态的**复合状态**           |
| **管理重心** | 统领性、标准定义    | 适配性、情境参数              |
