## 内容
**Datasmith** 是虚幻引擎（Unreal Engine）生态中最重要的"桥梁"技术。它的核心任务是解决 **"从 CAD/BIM 到 UE"** 的无损数据转换问题。

在没有 Datasmith 之前，工程师需要手动导出` FBX`，这会导致模型掉材质、坐标错乱、丢失层级结构。Datasmith 的出现让这个过程从"数周手动优化"缩短到了"一键导入"。

### 1. Datasmith 的核心能力

- **保留元数据（Metadata）**：这是数字孪生的关键。它不仅导入几何体，还会带入 BIM 里的信息（如 Revit 里的墙体材料、厚度、构件 ID、厂家信息等）。
- **自动处理材质映射**：它能自动将 CAD 软件里的材质（如 Revit 的玻璃、木材）智能转换为 UE 的高品质 PBR 材质。
- **保持层级与坐标**：导入后的场景结构与原始设计软件完全一致，方便在 UE 里进行逻辑编写。
- **同步更新（Direct Link）**：支持**实时联动**。你在 Revit 或 Rhino 里移动一个沙发，UE 里的模型会即时跟着移动，无需重复导出。

### 3. 支持的软件阵营（主要分类）

Datasmith 通过插件形式支持几乎所有主流的工业设计软件：

| 领域         | 支持软件                                | 协同价值                         |
| ---------- | ----------------------------------- | ---------------------------- |
| **BIM/建筑** | **Revit**, Archicad, SketchUp       | 将静态设计图转化为可交互、带实时光影的虚拟现场。     |
| **高级建模**   | **Rhino**, 3ds Max                  | 完美支持 Rhino 的复杂曲面和草图大师的轻量化模型。 |
| **机械/工业**  | **SolidWorks**, **CATIA**, Inventor | 导入高精度零件，用于装配模拟和产品展示。         |
| **基础设施**   | Civil 3D, IFC                       | 处理大规模地理信息和标准 BIM 数据交换格式。     |

### 3. 工作流：如何使用 Datasmith？
1. **安装插件**：在你的源软件（如 Revit）中安装 Datasmith 导出插件。
2. **导出/连接**：
    - **导出文件**：生成 `.udatasmith` 格式文件及其关联资源包。
    - **Direct Link**：开启实时连接模式（类似于两个软件间的"热线"）。
3. **UE 导入**：在 Unreal Engine 中点击 Datasmith 图标，选择文件或连接源。
4. **自动优化**：导入时，UE 会自动提示是否需要启用 Nanite（解决面数过高问题）或 Lightmass 烘焙配置。
### 4. Datasmith vs. FBX/OBJ

- **FBX**：仅传输几何体和基础贴图，丢失所有 BIM 属性信息，层级混乱。
- **Datasmith**：**数据+几何+属性**的全量同步。它是目前构建 **LOD 400-500 数字孪生** 的标准路径。

### 5. 在数字孪生中的实际应用

如果你要用 **UE 做数字孪生**，Datasmith 是你的第一步：

- **Revit → Datasmith → UE**：获取建筑骨架和设备参数。
- **IoT 数据 → MQTT → UE**：将实时传感器数据挂载到 Datasmith 导入的构件 ID 上。
- **最终成果**：一个既长得美（UE 渲染），又懂业务（BIM 属性），还能实时动（IoT 驱动）的数字孪生。
