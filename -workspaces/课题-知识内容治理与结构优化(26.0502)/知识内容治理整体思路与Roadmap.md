# 知识内容治理整体思路与Roadmap

**结论**：本方案的底层逻辑不是“重新整理目录”，而是建立一套知识生命周期治理机制：课题层负责把复杂问题编译清楚，知识层负责把稳定结论长期复用。

## 整体思路

| 层面   | 核心判断                                      | 对应动作                                    |
| ---- | ----------------------------------------- | --------------------------------------- |
| 底层路基 | 检索速度优先；一棵树一张嘴；MOC 是地图；证据留在课题层             | 先定义治理原则，再调整目录                           |
| 双层架构 | HXC Vault 做领域沉淀，Karpathy/llm-wiki 思路做课题编译 | `03-领域/` 管稳定知识，`---/-workspaces/` 管深度课题 |
| 转层机制 | 转层不是搬文件，而是提炼稳定结论                          | 用稳定性、泛化性、可复用性、证据充分性、归口清晰五项评分            |
| 实施路线 | 先修入口和边界，再迁移深度课题，最后建立复盘闭环                  | 重建 MOC、概念归口、课题筛选、编译层建设、知识回流、复盘治理        |
