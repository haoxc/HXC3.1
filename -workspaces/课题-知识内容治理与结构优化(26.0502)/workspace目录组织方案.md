# workspace 目录组织方案

**结论**：`---/-workspaces/` 采用四区结构——元管理层（`--进度管理`、`-backlogs`）用前缀做排序隔离，主题课题群按治理对象吸收 95% 内容，组织课题群处理跨主题的公司级课题，根层仅作临时过渡。D2 迁移（1,395 文件）按此结构分批归入。

## 现状问题

| # | 问题 | 表现 |
|---|------|------|
| 1 | 主题目录空壳 | 5 个主题 MOC（管理课题/AI课题/教学课题/设计课题/项目治理课题）下面零课题 |
| 2 | 散落课题未归类 | `课题-UE引擎/`、`课题-gbrain/`、`课题-知识内容治理/` 裸放在根 |
| 3 | 特殊目录混入 | `--进度管理`、`-backlogs` 用前缀做排序，属于元管理层而非主题 |
| 4 | 命名越界 | `领域-智能体调研/` 内容为课题过程，命名却像知识层 |

## 四区结构

```
---/-workspaces/
│
├── ■ 元管理层（-- / - 前缀排最前，固定 2 个）
│   ├── --进度管理/         看板/履约追踪/迭代进度
│   └── -backlogs/          积压课题/灵感/待启动
│
├── ■ 主题课题群（按治理对象分，吸收 95% 内容）
│   ├── 管理课题/
│   ├── AI课题/
│   ├── 教学课题/
│   ├── 设计课题/
│   └── 项目治理课题/
│
├── ■ 组织课题群（单一组织 ≥ 3 个课题时启用）
│   └── 止善(ZS)-课题清单/   已有
│
└── ■ 根层过渡（≤ 3 个，整理完 72h 内归入主题）
    └── 课题-xxx/            整理毕 → 移入主题目录
```

### 各区的判定与约束

| 区域 | 触发条件 | 数量上限 | 归入规则 |
|------|----------|:---:|------|
| 元管理 | 看板追踪、backlog 积压 | 固定 2 | `--`/`-` 前缀区分，永不归入主题目录 |
| 主题课题群 | 任何新建课题 | 不限 | 按治理对象归入对应主题，一个课题一个子目录 |
| 组织课题群 | 单一组织 ≥ 3 个课题 | 不限 | 课题仍可 Wikilink 二重链接到主题目录 |
| 根层过渡 | 批量迁移中的临时态 | ≤ 3 | 整理完毕 72h 内归入主题或组织群 |

## 主题目录判定表

```
课题治理对象          → 主题目录
──────────────────────────────────
经营管理/绩效/组织     → 管理课题/
Agent/模型/RAG/AI工具  → AI课题/
课程设计/教学/学习材料  → 教学课题/

Vault治理/skill/规范   → 项目治理课题/
单一组织≥3个课题       → {组织简称}-课题清单/（或新建）
```

## 命名规范

| 对象 | 格式 | ✓ | ✗ |
|------|------|------|------|
| 主题目录 | `{主题}课题/` | `AI课题/` | `AI相关/`、`01-AI/` |
| 课题目录 | `课题-{关键词}({日期})/` | `课题-战略指标量化(26.0502)/` | `战略指标/`、`调研-xxx/` |
| 组织目录 | `{组织简称}-课题清单/` | `止善(ZS)-课题清单/` | `止善/` |
| 根过渡 | `课题-{名称}/` | `课题-跑通gbrain流程/` | `gbrain/` |

- 主题目录默认不加日期（创建时无法判断短期还是长期）
- 课题目录日期仅在明确为短期冲刺/阶段任务时加

## D2 迁移映射

D2 内容全部来自 `01-我的/`，按治理对象映射到主题目录：

| 来源 | .md 数 | 映射目标 | 说明 |
|------|:---:|------|------|
| `常工院/` | 706 | `教学课题/` 下按课程拆子目录 | 教育项目 |
| `项目/` | 568 | 按内容分入管理/AI/教学等主题 | 混合内容，需逐项判定 |
| `工作室/` | 121 | `设计课题/` 或 `项目治理课题/` | 需看具体项目性质 |

**迁移策略**：分步执行——
1. 先在目标主题目录下建课题目录骨架
2. 每次搬一个来源：工作室 121 → 项目 568 → 常工院 706
3. 每步后运行 `generate_vault_index.py` 重建索引

## 边界：不做的事

- 不新建编号体系（`01-xx`）——主题名已足够语义化
- 不把元管理目录归入主题目录——前缀排序功能独立
- D2 迁移不分流到 `03-领域/`——先全量入 workspace，后续按 [[课题知识转入领域知识的判定标准]] 逐条评估
