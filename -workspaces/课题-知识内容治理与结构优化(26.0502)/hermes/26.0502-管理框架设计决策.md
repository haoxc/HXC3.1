# 管理框架设计决策

**决策日期**：2026-05-02
**状态**：设计中，未落地
**参与者**：用户（Vault 所有者）+ Hermes Agent

## 背景

Vault 知识管理面临内容治理课题——管理学科目录结构存在重量失衡、概念重复、边界模糊等问题。用户要求不限于管理学科个案，建立跨领域通用的目录管理框架。

## 核心决策

### 决策 1：采用三层管理框架

| 层   | 类比  | 载体               | 可校验性               |
| --- | --- | ---------------- | ------------------ |
| 声明层 | 宪法  | vault-rules.yaml | 机器可校验（YAML schema） |
| 规则层 | 法律  | 规则文档（含判据 + 反例）   | 人 + Agent 共识       |
| 流程层 | 执行  | skill + 角色       | 产出物校验              |
|     |     |                  |                    |
|     |     |                  |                    |

**理由**：已有材料天然分层——vault-rules.yaml 是声明层，CLAUDE.md / 课题目录建构规则 / 课题→领域转移标准是规则层，note-crafter / pjx-common-auditor 是流程层。框架不是新建，是串连。

**来源**：已有 vault-rules.yaml v1.0.0 + CLAUDE.md + 课题目录建构规则 + 课题→领域转移标准 + note-crafter skill + pjx-common-auditor skill。

### 决策 2：目录规划七原则

| #   | 原则      | 来源                       | 含义                                                  |
| --- | ------- | ------------------------ | --------------------------------------------------- |
| 1   | 归属优先    | vault-rules.yaml 顶层目录角色  | 内容先判归属（03-领域 vs 06-商业 vs 07-治理），再定深度                |
| 2   | 检索优先    | CLAUDE.md 底层逻辑           | 标题决定搜不搜得到 → 文件名 = 核心主题词，不加冗余前缀                      |
| 3   | 一棵树一张嘴  | CLAUDE.md 底层逻辑           | 每个概念只在一处详细解释，其他地方只 Wikilink                         |
| 4   | 课题-知识分离 | 双层架构（HXC vs Karpathy 对比） | 课题（workspace）≠ 知识页（领域），前者是过程产物，后者是稳定输出              |
| 5   | MOC 是地图 | vault-rules.yaml moc 节   | MOC 只做导航（≤20 行正文），禁止写成文章                            |
| 6   | 规模敏感    | vault-rules.yaml 阈值体系    | 笔记 >80 行考虑拆分，MOC >40 链接考虑拆分，目录 note_count >500 考虑拆分 |
| 7   | 改动最小    | 管理学科整改计划风险控制             | 不一次性大搬家、先修入口再迁移、旧页面不删除先归档                           |

**状态**：7 条原则已提取串联，但尚未展开为完整决策模型（缺每条的判据、适用场景、反例、优先级排序）。

### 决策 3：scope 边界——先框架后落地

当前阶段只讨论一级结构规划设计（声明层/规则层/流程层的关系与边界），不涉及 skill 修改、脚本编写、目录迁移等执行操作。

**理由**：用户明确指令——「先只说架构不落地」。框架未稳定前动手执行会产生返工。

### 决策 4：四角色框架（已扩展为 7 角色）

初始 4 角色（Note Crafter / Topic Architect / Knowledge Transfer / Vault Steward）在 Task 2 中扩展为 7 角色完整定义：

| 角色                    | 职责                             | 对应 skill                | 状态         |
| --------------------- | ------------------------------ | ----------------------- | ---------- |
| Material Collector    | 捕获原始材料（链接/摘录/观察）               | 无（人类主导）                 | 已定义        |
| Note Crafter          | 格式化笔记——命名/归档/frontmatter/挂 MOC | note-crafter            | 已有         |
| Knowledge Synthesizer | 课题层深度编译——对比/辨析/形成结论            | 无                       | 缺口         |
| Knowledge Transfer    | 课题→领域转移判定+提炼执行                 | 无                       | 缺口         |
| Vault Steward         | MOC 维护/链接修复/存量审计/过期归档          | pjx-common-auditor（偏审计） | 缺 MOC 维护功能 |
| Topic Architect       | 目录结构设计/MOC 分组策略/跨域关系（横切）       | 无                       | 缺口         |
| Knowledge Indexer     | 建索引/向量化/提供搜索（基础设施）             | gbrain                  | 已有         |

**详细定义 →** `知识管理角色与流程边界.md`
