## 阻塞：macOS 26.3 PGLite WASM Bug

### 症状

任何需要初始化 PGLite 的 gbrain 操作均崩溃：

```
PGLite failed to initialize its WASM runtime.
This is most commonly the macOS 26.3 WASM bug: https://github.com/garrytan/gbrain/issues/223
Original error: Aborted(). Build with -sASSERTIONS for more info.
```

### 已验证的尝试

| 方法 | 结果 |
|------|------|
| `bun run src/cli.ts serve` | ✗ WASM crash |
| `bun run src/cli.ts sync --help` | ✗ WASM crash |
| `bun run src/cli.ts doctor` | ✓ 文件系统模式（不依赖 PGLite） |
| Node.js v22.22.1 | ✗ TypeScript syntax error（参数属性不支持） |
| `git pull` to v0.25.0 | ✗ 无修复（PGLite 版本未变，仍 0.4.3） |
| `bun install` 重装依赖 | ✗ 同版本，无改善 |

### 已知信息

- Issue #223 确认：macOS 26.3 + Bun 1.3.11 + PGLite 0.4.4（本项目 0.4.3 同样受影响）
- "Pre-upgrade processes continue to work" — 升级前启动的进程不受影响
- 无已知 workaround（截至 2026-05-02 检查）

### 历史 MCP 连接

本次会话初期，gbrain MCP server 可正常响应（`get_stats`、`doctor` 均成功）——因该进程在 macOS 升级前启动。`kill -9` 杀掉卡住的 sync 后，server 进程随之退出，之后无法重新启动。

## 可行的前进路径

### 方案 A：等待 gbrain 修复 PGLite WASM

- 关注 Issue #223
- PGLite 上游修复后，`bun install` 更新依赖即可
- 不确定性高

### 方案 B：重启 macOS

- Issue 评论中有人提到重启可能清除 WASM 缓存状态
- 未验证，但不费时间
- 可尝试

### 方案 C：切换到 Postgres 后端

- gbrain 支持 `db.type: postgres`，绕过 PGLite WASM
- 需要本地 Postgres 服务
- 配置改动：`~/.gbrain/config.json` 中 `db.type` 和 `db.url`

### 方案 D：降级 Bun

- 安装 Bun < 1.3.11 版本
- Issue #223 未确认具体起作用的版本
