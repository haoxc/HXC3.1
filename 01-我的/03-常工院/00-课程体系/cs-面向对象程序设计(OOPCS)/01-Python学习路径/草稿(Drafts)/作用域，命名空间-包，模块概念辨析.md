## 三、模块（Module）：一个 `.py` 文件 = 一个命名空间 + 全局作用域

- 每个 `.py` 文件就是一个**模块**。
- 模块有自己的**全局作用域**，也就是它的**命名空间**。
- 导入模块时，就相当于引入了它的命名空间。

### 例子：

```python
# colors.py
primary = "#FF0000"

def get_secondary():
    return "#00FF00"

# main.py
import colors
print(colors.primary)        # 访问 colors 模块的命名空间
print(colors.get_secondary())
```

> ✅ 模块 = 一个独立的创作空间，自带一套工具和颜色。

## 🔗 四者关系总结

|概念|本质|与其它概念的关系|
|---|---|---|
|**命名空间**|名字 → 对象的映射（字典）|模块、函数、类都有自己的命名空间|
|**作用域**|代码中名字的“可见范围”规则|决定了在某处能访问哪些命名空间|
|**模块**|一个 `.py` 文件|拥有一个全局命名空间，构成一个作用域单元|
|**包**|包含模块的目录（带 `__init__.py`）|是模块的容器，本身也是命名空间|

## ✅ 实践建议（给数媒艺术生）

1. **写小项目时**：把常用函数放进 `mytools.py`（模块），用 `from mytools import draw_star`。
2. **做大项目时**：用包组织代码，比如：
    
    ```
    generative_art/
    ├── __init__.py
    ├── shapes.py      # 几何图形
    ├── palettes.py    # 色彩方案
    └── animations.py  # 动态效果
    ```
    
3. **遇到名字冲突**：用 `dir(module)` 查看模块里有什么，用 `module.name` 明确指定命名空间。
