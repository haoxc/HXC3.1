
## 核心定义

| 概念         | 本质                 | 数媒生通俗理解        | 最小单位   |
| ---------- | ------------------ | -------------- | ------ |
| Package（包） | 带`__init__.py`的文件夹 | 「功能文件夹」，归类相关代码 | 文件夹    |
| Library（库） | 一组功能相关的包/模块的集合     | 「完整工具集」，解决一类需求 | 多个包/模块 |

## 数媒场景举例
### 1. Package（包）：数媒素材处理的“功能文件夹”
比如你要做「数媒素材处理」相关代码，会建这样的文件夹结构：
```
image_tools/  # 这是一个Package（包）
    __init__.py  # 包的标识文件（空文件也可以）
    image_info.py  # 我们写的ImageInfo类模块
    batch_process.py  # 批量处理素材的模块
color_tools/  # 另一个Package（包）
    __init__.py
    color.py  # 我们写的Color类模块
    palette.py  # 调色板模块
```
- 包的核心作用：把功能相关的`.py`文件（模块）放在同一个文件夹，避免代码混乱；
- 数媒场景：比如`image_tools`包专门管图片素材，`color_tools`包专门管颜色，分工清晰。

### 2. Library（库）：数媒设计的“完整工具集”
比如你熟悉的`Pillow`（PIL）就是一个Library（库）：
- 它包含多个Package（包）：`PIL.Image`、`PIL.ImageDraw`、`PIL.ImageFilter`等；
- 每个包下有多个模块：比如`PIL.Image`包下有处理图片的核心模块；
- 数媒场景：Pillow库能解决“图片解析、格式转换、添加滤镜”等所有图片处理需求，是一套完整的“设计工具集”。

## 关键区别
1. **包（Package）是“组织方式”**：你自己写代码时，用包把`ImageInfo`、`Color`等类按功能归类，方便后续复用；
2. **库（Library）是“现成工具”**：比如Pillow、turtle、matplotlib都是库，数媒生直接导入使用（`import PIL`），不用自己写核心功能；
3. **模块（Module）是基础**：单个`.py`文件（比如`image_info.py`）是模块，包是“模块的文件夹”，库是“包/模块的集合”。

## 总结
- 写代码时：你建的`image_tools`文件夹（带`__init__.py`）= 包；
- 用工具时：你装的`pip install pillow`= 库；
- 核心逻辑：模块（.py）→ 包（文件夹）→ 库（工具集）。
