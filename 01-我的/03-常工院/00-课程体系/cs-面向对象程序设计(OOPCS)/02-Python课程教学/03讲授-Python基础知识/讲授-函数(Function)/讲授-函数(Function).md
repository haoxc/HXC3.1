## 概述
- **定义**：函数是封装好的，可重复使用的，用来实现单一，或相关联**功能**的代码段。
- **作用**： 
	- **代码复用**：相同功能无需重复编写，调用函数即可。
	-  **抽象封装**：隐藏实现细节，只需关注 “输入” 和 “输出”（如调用`calculate_area`时，无需关心内部如何计算）。
	- **模块化**：将复杂程序拆分为多个函数，逻辑更清晰，便于维护。

> 🎨 Python 函数
> - 你的“数字艺术创作工具”
> - **函数 = 一个可重复使用的创作动作**
> - **函数就是你的“代码笔刷”**

## 内容
### 基本结构
```python
def 函数名(参数1, 参数2=默认值):  # 定义工具名称和可调参数
    """函数说明（像动作的备注：这个工具做什么）"""
    # 函数体：具体操作步骤（调亮度、加边框等）
    处理步骤1
    处理步骤2
    return 处理结果  # 可选：返回处理好的内容（如调整后的图片）
```

####  可变参数
可变参数是 Python 中用于处理 “参数数量不确定” 场景的特殊参数形式，允许函数接收任意数量的输入，主要分为两种：`*args` 和 `**kwargs` 

-  `args`：接收任意数量的位置参数
	- **作用**：收集所有未被明确匹配的**位置参数**，打包成一个**元组**（tuple）传递给函数。
	- **语法**：在参数名前加 `*`（约定用 `args` 作为参数名，也可使用其他名称，但 `args` 是惯例）。

**示例：计算多个数字的总和**
```python
def sum_numbers(*args):
    """计算任意数量数字的总和"""
    print("接收的参数（元组）：", args)  # 查看args的结构
    return sum(args)  # 调用系统函数, 直接对元组求和

# 调用函数：传入不同数量的参数
print(sum_numbers(1, 2, 3))  # 传入3个参数，输出：6
print(sum_numbers(10, 20, 30, 40, 50))  # 传入5个参数，输出：150
print(sum_numbers())  # 传入0个参数，输出：0（sum对空元组返回0）
```

-  `**kwargs`：接收任意数量的关键字参数
	- **作用**：收集所有未被明确匹配的**关键字参数**（即 “参数名 = 值” 形式），打包成一个**字典**（dict）传递给函数。
	- **语法**：在参数名前加 `**`（约定用 `kwargs` 作为参数名，也可使用其他名称，但 `kwargs` 是惯例）。

#### 作用域 
- 函数内部定义的变量 = **只在该工具内部有效**（局部变量）
- 函数外部定义的变量 = **整个画布可见**（全局变量）】
**示例**：
```python
background = "black"  # 全局变量（整个作品背景）
def draw_star():
    size = 20  # 局部变量（只在画星星时用）
    # ... 画星星的代码
    print(f"正在用背景色 {background} 画星星")  # 可以访问全局变量

# print(size)  # ❌ 报错！size 只在 draw_star 内部存在
```

>✅ **原则**：尽量用参数传递数据，少用全局变量（避免“调色盘混乱”）

## 实践

**示例 1**： 定义一个 “给图片加边框” 的函数：
```python
import cv2  # 导入OpenCV库，用于图像处理  
def add_border(image_path, width=10, color=(255, 255, 255)):  
    """  
    给图片添加边框的函数  
    参数：  
        image_path：字符串，待处理图片的路径（相对路径或绝对路径）  
        width：整数，边框的宽度（默认10像素）  
        color：元组，边框颜色（BGR格式，默认白色(255,255,255)）  
    返回：        带边框的图像矩阵（numpy数组），若读取图片失败则返回None  
    """   
    # 读取图片：cv2.imread返回图像矩阵，失败时返回None  
    image = cv2.imread(image_path)  
    bordered_image = cv2.copyMakeBorder(image,width,width,width,width,cv2.BORDER_CONSTANT,color)  
    return bordered_image  # 返回添加边框后的图像  
  
# 调用函数：给指定图片添加边框  
result_image = add_border("bg_landscape_Everest_zmlm.jpg")  
# 若处理成功，保存结果图片  
if result_image is not None:  
    # 保存带边框的图片到当前目录，文件名为"bordered_image.jpg"  
    cv2.imwrite("bordered_image.jpg", result_image)  
    print("边框添加成功，结果已保存为 bordered_image.jpg")
```
参数是函数的 “输入选项”，让同一个函数能处理不同需求（比如同一 “加边框” 动作，可改边框宽度）。

| 参数类型  | 数媒类比           | 示例（加边框函数）                                                 |
| ----- | -------------- | --------------------------------------------------------- |
| 位置参数  | 必选选项（如边框必须有宽度） | `add_border(img, 10)` → 宽度 10（按顺序传值）                      |
| 关键字参数 | 可选选项（指定名称修改）   | `add_border(img, color=(0,0,0))` → 黑色边框 [[py-缺省参数(None)]] |
| 默认参数  | 预设选项（不调就用默认值）  | 不传 `width` 时，自动用默认值 5                                     |
|       |                |                                                           |
> 📌 就像 PS 动作里的 “可调选项”

## 建议和技巧

1. **命名清晰**：`draw_mandala()` 比 `func1()` 好一万倍！
2. **功能单一**：一个函数只做一件事（如“画花瓣”，不要又画花瓣又调色）
3. **多写注释**：用 `"""` 写清楚这个函数是干嘛的
4. **先写再用**：把常用动作封装成函数，主程序只负责“组合创作”
