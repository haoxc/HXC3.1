#  Python `for` 循环

- 自动化重复操作的核心工具
---
## 为什么需要 `while` 循环？

**核心问题**：当重复次数不确定，但知道 “停止条件” 时，如何自动重复操作？
-  逐帧检测~~动~~画，直到某一帧的亮度达标（不知道要检测多少帧，但**知道** “`亮度≥200 就停`”）
- 批量裁剪图片，直到文件夹里的图片处理完（不知道有多少张，但**知道** “`没图片了就停`”）
 
> [!NOTE] **while 循环的作用（适用场景）**
>  → 只要某个条件成立，就一直重复执行操作
>  → 适合 “次数不确定，但有明确停止条件” 的场景

---
## 核心逻辑

**假定场景**：
「「调整图片亮度」—— 只要图片亮度＜150（条件），就继续调亮（重复操作）
```python
while 条件:  #条件是布尔值（True/False）
    重复执行的操作  # 缩进！
    # （通常需要修改条件中的变量，避免无限循环）
```

**基本文法**：
```python
whie 循环条件:
    循环体操作1  # ← 必须缩进（4个空格）
    循环体操作2
    变更循环条件
    ...
```
>  `循环条件`：控制是否进入循环内部

> [!Warning] 关键
> - 循环中必须有代码让 “**循环条件最终变为 False**”，否则会无限重复（像卡死的软件）

**流程图**：
```  mermaid
flowchart TD
    A@{ shape: circle, label: "开始" } --> C{循环套件}
    C -->|是| D[循环体]
    C -->|否| E@{ shape: dbl-circ, label: "结束" }
    D --> |继续| C
 ```

## 实战示例

### 示例 1：遍历动画帧，直到某帧亮度≥200 时停止检测（假设亮度数据存在列表中）

```python
frames_brightness = [120, 150, 170, 210, 230]  # 各帧亮度
current_frame = 0  # 当前检测的帧索引

# 只要当前帧亮度＜200，就继续检测
while frames_brightness[current_frame] < 200:
    print(f"第{current_frame}帧亮度不足（{frames_brightness[current_frame]}），继续检测...")
    current_frame += 1  # 切换到下一帧（关键：修改条件变量）

print(f"找到目标帧：第{current_frame}帧，亮度{frames_brightness[current_frame]}")
```

## 总结
**与 for 循环的区别**

| 循环类型     | 适用场景         | 数媒举例                |
| -------- | ------------ | ------------------- |
| for 循环   | 次数确定（遍历序列）   | 处理固定 20 帧动画、10 张图片  |
| while 循环 | 次数不确定（靠条件停止） | 检测到合格帧才停止、等待用户输入后继续 |
> [!Error] 关键注意点
> 1. **避免无限循环**：必须在循环内修改条件变量（如`current_frame += 1`），否则条件永远成立，程序卡死。
> 2. **条件是布尔值**：条件可以是比较表达式（如 `x < 10`）、变量（如 `is_running`）等，结果为 True/False。
