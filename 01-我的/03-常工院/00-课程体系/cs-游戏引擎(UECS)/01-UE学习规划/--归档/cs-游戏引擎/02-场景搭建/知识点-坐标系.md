Unreal Engine（UE）的坐标系统是其 3D 空间计算、物体定位、渲染和交互的基础，理解它对场景搭建、蓝图逻辑、模型导入等操作至关重要。 ：

### 一、基本坐标系：左手坐标系

UE 采用左手坐标系（Left-Handed Coordinate System），这是区分于 Blender（右手）等软件的核心特征。判断方式：伸开左手，让拇指、食指、中指两两垂直

![[e37aef6173e748c5e32b80f39f0f93ee_MD5.png]]

  

- 拇指指向 X 轴正方向
    
- 食指指向 Y 轴正方向
    
- 中指指向 Z 轴正方向
    

### 二、轴方向定义（世界空间）

在 UE 的世界空间（World Space） 中，三个轴的方向是固定的，所有物体的位置最终都以世界空间坐标为基准：

![[a7c0b06c8d204096351facde81e799f0_MD5.png]]

- X 轴：水平向右（East，东方向）
    
- Y 轴：水平向前（North，北方向）—— 物体 “前进” 的默认方向（如角色移动、摄像机朝向）
    
- Z 轴：垂直向上（Up，上方向）—— 高度方向
    

### 三、核心坐标空间

UE 中存在多种坐标空间，不同空间的原点和轴方向不同，用途也不同：

|   |   |   |   |
|---|---|---|---|
|坐标空间|原点位置|轴方向定义|核心用途|
|世界空间（World Space）|场景的全局原点（0,0,0）|X 右、Y 前、Z 上（固定不变）|确定物体在整个场景中的绝对位置（如地图上的坐标）。|
|局部空间（Local/Object Space）|物体的 pivot 点（枢轴点）|以物体自身为基准：X 轴向右（物体 “右侧”）、Y 轴向前（物体 “前方”）、Z 轴向上（物体 “顶部”）。|控制物体自身的旋转、缩放（围绕 pivot 点），或局部范围内的子物体定位（如角色的手相对于身体的位置）。|
|父空间（Parent Space）|父物体的 pivot 点|继承父物体的局部空间轴方向（子物体坐标相对于父物体）。|层级化物体定位（如 “汽车 - 车轮”：车轮坐标相对于汽车 pivot 点）。|
|视图空间（View/Camera Space）|摄像机的 pivot 点|X 轴向右（屏幕水平右）、Y 轴向上（屏幕垂直上）、Z 轴向后（摄像机 “后方”，视线方向为 - Z 轴）。|渲染计算（如判断物体是否在摄像机视野内）、UI 与 3D 物体交互（如射线检测）。|
|屏幕空间（Screen Space）|屏幕左上角（2D）或中心（3D 转 2D）|X 轴向右（屏幕宽度）、Y 轴向下（屏幕高度），范围通常为像素值（如 1920x1080 屏幕的 X:0~1920，Y:0~1080）。|2D UI 布局（如 UMG 控件位置）、3D 物体投影到屏幕的坐标计算（如 “点击 3D 物体” 的屏幕坐标判断）。|

### 四、旋转规则（欧拉角与轴向）

UE 中物体的旋转（Rotation）基于欧拉角，用 “俯仰角（Pitch）、偏航角（Yaw）、翻滚角（Roll）” 描述，分别对应围绕三个轴的旋转：

![[c2c9db92358c629ded8d576a53c48d0a_MD5.png]]

- Yaw（偏航）：围绕 Z 轴 旋转（物体 “左右转向”，如角色转身）。
    
    - 正值：顺时针旋转（从顶视图看）；负值：逆时针旋转。
        
- Pitch（俯仰）：围绕 X 轴 旋转（物体 “抬头 / 低头”，如摄像机上下看）。
    
    - 正值：向上旋转（视线向上）；负值：向下旋转。
        
- Roll（翻滚）：围绕 Y 轴 旋转（物体 “侧翻”，如飞机倾斜）。
    
    - 正值：向右翻滚（从正前方看）；负值：向左翻滚。
        

  

> 注意：旋转顺序默认是 Z→X→Y（Yaw→Pitch→Roll），可在蓝图 / 代码中修改旋转顺序，但默认顺序是最常用的。

### 五、坐标单位

UE 中1 个单位 = 1 厘米（cm），这是模型导入和尺寸匹配的关键。例如：

- 一个 “180cm 高的角色”，在 UE 中 Z 轴尺寸应设置为 180 单位。
    
- 从 Blender（默认单位为米）导入模型时，需将模型缩放 100 倍（1 米 = 100 厘米），否则会 “变大 100 倍”。
    

### 六、与其他软件的差异（避坑点）

|   |   |   |   |
|---|---|---|---|
|软件|坐标系类型|轴方向差异（与 UE 对比）|旋转轴向差异|
|Unity|左手坐标系|Y 轴为 “向上”，Z 轴为 “向前”（UE 是 Y 轴向前、Z 轴向上）。|Yaw 围绕 Y 轴，Pitch 围绕 X 轴（与 UE 完全不同）。|
|Blender|右手坐标系|Z 轴为 “向前”，Y 轴为 “向上”（且是右手系，轴方向完全不同）。|旋转轴向与 UE 无直接对应，导入需勾选 “应用旋转”。|
|Maya|右手坐标系|Y 轴为 “向前”，Z 轴为 “向上”（但左右手系不同）。|旋转顺序和轴向定义与 UE 差异较大。|

  

解决模型方向错误：导入时在 UE 的 “导入设置（Import Settings）” 中勾选 “Convert Scene”，UE 会自动转换坐标适配自身系统。

### 总结

UE 坐标系统的核心是：**左手坐标系****，X 右、Y 前、Z 上**，围绕不同空间（世界 / 局部 / 视图）定义坐标，并通过 “Pitch（X）、Yaw（Z）、Roll（Y）” 控制旋转。理解这些规则能避免模型导入错位、角色移动方向错误、UI 与 3D 交互失效等问题，是 UE 开发的基础。
