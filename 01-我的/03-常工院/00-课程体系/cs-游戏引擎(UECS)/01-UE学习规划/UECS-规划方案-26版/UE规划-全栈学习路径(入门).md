# Unreal Engine 全栈学习与实践路线图

---
## 1. 核心知识版图 (Knowledge Map)

要掌握 Unreal Engine (UE)，需要将复杂的工具集拆解为四个核心维度。
### A. 视觉表现层 (Visuals)

* **Lumen & Nanite**: 掌握第五代引擎的核心——动态全局光照与虚拟化微多边形几何体。
* **材质编辑器 (Material Editor)**: 基于物理的渲染 (PBR) 流程，理解着色器逻辑。
* **特效系统 (Niagara)**: 学习如何构建粒子系统、流体模拟及物理交互。
### B. 逻辑交互层 (Logic)

* **蓝图系统 (Blueprints)**: 掌握节点式编程，理解事件驱动架构。
* **C++ 核心**: 了解底层 API、内存管理以及性能瓶颈的优化方案。
* **增强输入 (Enhanced Input)**: 跨平台设备的输入适配逻辑。

### C. 环境构建层 (World Building)

* **地貌与植被 (Landscape & Foliage)**: 大规模地形编辑及程序化生成。
* **世界分区 (World Partition)**: 学习大世界加载优化技术。
* **关机设计 (Level Design)**: 空间构图与灯光氛围营造。

### D. 性能与输出层 (Optimization)

* **性能分析 (Profiling)**: 使用 GPU Visualizer 和 Insights 查找掉帧原因。
* **渲染输出 (Movie Render Queue)**: 高质量影视级渲染配置。

---

## 2. 知识关联关系 (Relationship Topology)

理解这些模块如何协同工作，是进阶为高手的关键：

| 核心组件              | 关联关系            | 实践意义                                 |
| ----------------- | --------------- | ------------------------------------ |
| **Actor (角色)**    | **蓝图 + 模型**     | 它是所有场景对象的基类，逻辑与视觉的载体。                |
| **材质 (Material)** | **参数集 + 蓝图**    | 通过蓝图动态修改材质参数，实现交互式视觉变化。              |
| **Nanite**        | **Lumen + 几何体** | 高精度的模型（Nanite）能提供更真实的反射与阴影反馈（Lumen）。 |
| **Data Asset**    | **逻辑 + 数据分离**   | 将配置数据与蓝图解耦，方便后期维护与多方案切换。             |

## 3. 聚焦实践：从“知道”到“做到”

### 第一阶段：原型构建 (Prototype)

* **目标**: 1周内实现一个可操作的第三人称基础场景。
* **要点**: 掌握 Actor 生命周期，跑通按键控制角色移动。

### 第二阶段：视觉打磨 (Polishing)

* **目标**: 构建一个影视级光影的小室场景。
* **要点**: 熟练应用 PPV (后处理体积) 和高清材质贴图。

### 第三阶段：系统整合 (System Integration)

* **目标**: 开发一个包含 UI 交互、任务系统、音效触发的完整 Demo。
* **要点**: 学习蓝图接口 (Interfaces) 和事件分发器 (Event Dispatchers)。

---

> [!TIP] 实践指南
> **“最小闭环原则”**：不要试图先看完所有视频教程再动手。每学习一个新节点（如：Timeline），立即在场景里做一个对应的交互开关（如：感应门）。这种“即学即见”的反馈能极大地增强你的神经记忆。

## 4. 成果达成 Checklist

* [ ] **独立打包**：能将项目打包为 Windows/Android 可执行文件，且无崩溃。
* [ ] **性能达标**：在目标平台上稳定运行（如 1080p/60fps）。
* [ ] **代码整洁**：蓝图逻辑清晰，无交叉连线，注释完整。
* [ ] **资源管理**：熟练使用重定向器，清理项目中的无用资产。
