# Unreal Engine 技术专题：纹理图集 (Texture Atlas)

---
## 1. 核心知识要点 (Key Knowledge Points)

在 Unreal Engine 中，**Atlas (图集)** 是一种将多个较小的纹理（Texture）合并到一张大纹理图中的技术。它是性能优化的核心手段之一。
- **减少 Draw Calls (绘制调用)**：这是 Atlas 的核心价值。渲染 100 个使用独立纹理的小物体需要 100 次绘制调用，但如果它们共享一张图集，理论上可以合并为极少数甚至 1 次调用。
- **纹理利用率 (Packing Efficiency)**：通过算法将不同尺寸的子图（Sub-textures）紧密排列，最大限度减少大图中的空白浪费。
- **UI 与 2D (Paper2D/UMG)**：在 UI 开发中，图集是必须项。将图标、边框、按钮合并，能显著提升界面加载速度。
- **纹理边距 (Padding/Bleeding)**：为了防止在远处（Mipmaps 触发时）相邻子图的颜色互相渗透，图集中的子图之间必须保留足够的像素间距。

---

## 2. 知识关联关系 (System Topology)

理解 Atlas 如何在引擎底层发挥作用，有助于你在开发中做出正确的性能权衡：

| **关联模块**                        | **协作逻辑**                 | **实践意义**                        |
| ------------------------------- | ------------------------ | ------------------------------- |
| **UV Mapping**                  | 模型表面的 UV 坐标需精确指向图集中的特定区域 | 模型制作时需提前规划 UV 空间或使用自动展 UV 工具。   |
| **Virtual Texturing (RVT/SVT)** | 可以看作是“动态的、超大型图集”         | 在 UE5 大世界场景中，RVT 逐渐取代了部分手动图集工作。 |
| **Material Instance**           | 多个物体使用同一个材质实例，并切换图集坐标    | 减少 Shader 切换开销，提升 GPU 处理效率。     |
| **Merge Actors**                | UE 内置工具，可将多个静态网格体及其纹理合并  | 用于优化复杂的关卡组件（如一堆杂乱的瓦砾）。          |

## 3. 聚焦实践：图集创建与应用流程

### 第一阶段：资源合并 (DCC 或 插件)
1. **外部制作**：使用 Substance Designer 或 Texture Packer 导出已合并的图集及对应的坐标数据。
2. **UE 内置合并**：
    - 选中多个 Actor。
    - 右键选择 `Merge Actors`。
    - 在设置中勾选 `Generate Diffuse/Normal/Roughness Maps` 并设定合并后的 `Atlas Size`。

### 第二阶段：材质偏移 (UV Offset)

- **逻辑**：通过材质球中的 `TextureCropping` 节点或手动使用 `Add` 和 `Multiply` 节点对 UV 进行缩放和平移。
- **应用**：确保模型只显示大图中的一小部分。

### 第三阶段：UI 图集优化 (Slate/UMG)

- 使用 **Sprite Actions -> Create Sprite**。
- 在 **Texture Group** 中设置为 `UI`。
- 引擎会自动处理这些 Sprite 在运行时的内存占用和合批逻辑。

---

> [!IMPORTANT] 实践指南
>
> 解决“接缝”与“漏色” (Bleeding)：
>
> 当你发现模型在远处出现奇怪的黑线或相邻图片的颜色时，通常是因为 Mipmap 导致的采样溢出。
>
> 建议：在创建图集时，子图之间至少预留 8-16 像素 的间距，并使用“边缘拉伸 (Edge Dilate)”技术填充这些间距，这样在低分辨率显示时依然能保持颜色纯净。

---

## 4. 成果达成 Checklist
- [ ] **Draw Call 监测**：使用 `stat engine` 命令，观察合并图集后 Draw Calls 是否有明显下降。
- [ ] **内存检查**：确保合并后的大图没有超过目标平台的纹理上限（如移动端建议控制在 2048x2048 或 4096x4096px）。
- [ ] **UV 对齐**：模型在缩放或旋转后，其显示的子图区域依然精准，无偏移现象。
- [ ] **Mipmap 兼容性**：在不同距离观察物体，边缘依然干净，没有产生采样闪烁或漏色。
