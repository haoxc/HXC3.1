## 🗺️ 总体架构

```
本地 LLM (Qwen3 14B via Ollama)
        ↓
UE5 Plugin 层（MCP / REST API）
        ↓
场景搭建 │ 蓝图编程调试 │ 动画驱动
```

---

## 第一阶段：蓝图编程辅助（最成熟，优先做）

### 推荐工具：**UE LLM Toolkit** + Claude Code / Cursor

这是一个纯 C++ 插件，通过 HTTP 暴露 37 个工具、200+ 操作，给 LLM 提供读写 UE 编辑器主要子系统的能力，支持读取蓝图图形、追踪动画状态机、检查 Montage Notifies，兼容 Claude Code、Cursor、Windsurf 等任何 HTTP 客户端，不需要 Python 运行时，无 MCP 锁定。

**执行步骤：**

```
1. 克隆插件：
   git clone https://github.com/ColtonWilley/ue-llm-toolkit

2. 将 Plugins/UELLMToolkit 放入你的 UE 项目 Plugins 目录

3. 编译项目，启动编辑器

4. 安装 Claude Code CLI（命令行）：
   npm install -g @anthropic-ai/claude-code

5. 在项目根目录执行：
   claude  ← 进入交互，让 AI 扫描项目建立 domain 文件（一次性）
```

**实际能做到的事：**

- "这个角色跳跃蓝图为什么不触发？帮我找 bug"
- "给我写一个`开门交互蓝图`"
- "分析这个动画状态机的转换条件"

## 第二阶段：场景搭建自动化

### 推荐工具：**Ultimate Engine Copilot（原 Ultimate Blueprint Generator）**
这个工具从简单蓝图生成器进化为全引擎 AI 助手，支持材质、3D 模型生成，提供一次性购买（$220），含完整 C++ 源码，支持本地 LLM 接入（2026年1月26日已添加），可使用免费 Gemini API Key 或本地 LLM，无需月租。

**本地 LLM 接入配置：**

```json
// 插件设置中填入：
{
  "provider": "ollama",
  "base_url": "http://127.0.0.1:11434/v1",
  "model": "qwen3:14b"
}
```

**能做到的事：**
- 输入 "创建一个有雾气的森林场景，包含岩石、树木和河流" → 自动摆放资产
- 一句话重构整个 UI 布局
- 场景灯光批量调整

## 第三阶段：动画驱动（MetaHuman / NPC）

### 推荐工具：**Convai** + 本地 LLM 代理

将 LLM 与高精度 3D Avatar 集成，开发者可以让玩家用自然语音与 NPC 对话，AI 理解上下文、记住背景故事并动态回应，其 NeuroSync 动画系统实时处理音频流，无需离线处理或预烘焙动画，在 UE 蓝图中添加 BP_ConvaiFaceSync 组件即可驱动 MetaHuman 面部表情与口型同步。

**对于非 NPC 动画（过场动画、序列）：**

使用 **DeepMotion**（动捕转动画）：

```
1. 录制视频（手机即可）
2. 上传 DeepMotion → 导出 FBX
3. 导入 UE5 → 绑定到 Skeleton
4. 用 LLM 生成 Animation Blueprint 逻辑
```

## ⚠️ 避坑指南

1. **蓝图无法直接被 LLM 读取**：`.uasset` 是二进制格式，必须通过 UE LLM Toolkit 这类插件先转 JSON 再喂给模型
2. **Qwen3 14B 上下文限制**：大项目建议用 Toolkit 的 domain 系统分批加载，token 使用量可降低约 70%
3. **本地模型不联网**：场景资产搜索建议配合 `FAB` 市场人工选取，AI 负责摆放和配置
4. **动画最难自动化**：复杂过场建议 AI 生成`状态机逻辑`，动捕数据仍需人工录制
