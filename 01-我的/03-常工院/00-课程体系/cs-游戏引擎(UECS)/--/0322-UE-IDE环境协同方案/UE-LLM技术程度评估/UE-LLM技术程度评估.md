## 核心原则：三类自动化模式

**模式1：分析型**（最稳定） LLM 读取现有的蓝图/代码/日志，给出诊断和修改建议。UE LLM Toolkit 能追踪动画状态机、检查 Montage Notifies、读取蓝图图形，AI 每次会话前加载 `domain 文件`，token 消耗降低约 70%，避免每次从头理解整个项目。

**模式2：生成型**（效果最显著） 输入需求描述，AI 输出蓝图/C++/配置。通过` MCP UE5` 服务器，LLM 可直接控制编辑器生成蓝图、函数、变量，并执行 Python 脚本，实现从自然语言到场景构建的全流程自动化。

**模式3：协同型**（长期价值最高） AI 最佳实践是把 LLM 当"需要指导的初级结对程序员"而非替代者——开发者提供架构决策，AI 负责实现细节，每完成一个功能点及时 commit，保持版本控制作为安全网。

## 对你项目的优先级建议

按投入产出比排序，建议从上往下依次落地：
1. `编译错误分析` → 几乎零配置，粘贴日志即可
2. `蓝图 Bug 诊断` → 装 UE LLM Toolkit 插件
3. `C++ 代码补全` → 配置 Rider + Qwen2.5-Coder
4. `场景资产摆放` → 接入 UnrealGenAI MCP
5. NPC 智能对话 → 集成 Convai 或 Cactus AI
