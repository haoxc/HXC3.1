## 内容

### 简介
OpenCode 的定位是：**面向专业开发者的、全流程可控的开源 AI 编程代理。**
- **交互形态：** 侧重于终端(Terminal)操作，支持 [[REPL交互模式]]，能够直接读取文件、运行测试并根据报错自动修复。
- **技术路线：** 不同于闭源产品的“黑盒”逻辑，OpenCode 强调 **透明性(Transparency)** 和 **可扩展性(Extensibility)**，允许用户自定义 Agent 的思考链路（Plan -> Execute -> Reflect）。

### 对比

| **维度**    | **OpenCode**                                                            | **竞品 (如 Claude Code, GitHub Copilot)** |
| --------- | ----------------------------------------------------------------------- | -------------------------------------- |
| **模型自由度** | **极高**。支持 75+ 供应商，可随时切换 Claude 3.7、GPT-4o 或**本地模型**(如 DeepSeek, Llama)。 | **受限**。通常绑定自家模型（如 Claude 或 GPT 系列）。    |
| **数据隐私**  | **强**。支持完全离线运行（结合 Ollama/LM Studio），代码不上传云端。                            | **弱**。核心逻辑和上下文通常需要上传至厂商服务器。            |
| **任务切换**  | **灵活**。支持在同一个任务中切换模型（如用 Claude 规划架构，用 GPT-4o 写具体逻辑）。                    | **单一**。单一任务通常只能由单一模型链完成。               |
| **成本控制**  | **按需付费**。直接调用 API 或使用本地模型，无强制月费。                                        | **订阅制**。通常为 $20+/月的固定费用。               |
| **开源生态**  | **可审计、可定制**。开发者可以修改 Agent 内部的 Prompt 和工具调用逻辑。                           | **黑盒**。用户无法得知 AI 为什么选择某种修改方案。          |
