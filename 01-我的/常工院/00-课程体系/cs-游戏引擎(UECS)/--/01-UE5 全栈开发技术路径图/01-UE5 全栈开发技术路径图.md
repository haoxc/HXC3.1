本路线图旨在将之前探讨的 **Nanite (几何)**、**PBR (材质)**、**HDRI (环境)** 以及 **Atlas/Mipmap (优化)** 串联成一个完整的生产闭环。

# UE5 全栈开发技术路径图

---

## 1. 核心知识版图：从理论到架构

### A. 资产构建层 (Asset Foundation)

- **高精度模型 (Nanite)**：突破多边形限制，聚焦于微多边形渲染。
- **物理材质标准 (PBR)**：建立基于物理的 Base Color, Roughness, Metallic 规范。
- **数据打包 (Atlas)**：利用图集减少 Draw Calls，提升 GPU 处理效率。

### B. 环境呈现层 (Environment Rendering)

- **动态光照 (Lumen)**：全局光照与间接光反射的实时处理。
- **环境模拟 (HDRI)**：通过基于图像的光照 (IBL) 快速对齐真实世界的光影数据。
- **大气系统 (Atmosphere)**：结合体积云、高度雾营造空间深度感。

### C. 性能底座层 (Optimization Base)
- **纹理流送 ([[Mipmap]])**：基于距离的采样优化，平衡显存占用与抗锯齿。
- **内存管理 (Streaming)**：确保 2^n 幂次方规范，激活流送池管理。
- **渲染分析 (Profiling)**：使用 GPU Visualizer 和 Stat 指令定位瓶颈。

## 2. 模块间的关联逻辑

| **技术模块**           | **核心关联**       | **协同产出**        |
| ------------------ | -------------- | --------------- |
| **Nanite + PBR**   | 几何细节与材质质感的物理对齐 | 影视级的资产表现力。      |
| **HDRI + Lumen**   | 全局环境基调与动态光影弹射  | 具备环境真实感的实时渲染。   |
| **Atlas + Mipmap** | 渲染合批与纹理多级采样的结合 | 支撑大规模复杂场景的流畅运行。 |

---

## 3. 实践路径：从入门到专家

### 阶段一：资产规范化 (Standardization)

- **目标**：掌握模型导入、RMA 打包材质、Mipmap 2 的幂次方原则。
- **达成成果**：单个资产在任何光照环境下均表现一致，且显存开销可控。

### 阶段二：场景集成化 (Integration)

- **目标**：配置 HDRI Backdrop，对齐主光源，开启 Nanite 与 Lumen。
- **达成成果**：构建一个光影自然、细节丰富的静态/动态环境。

### 阶段三：系统优化化 (Optimization)

- **目标**：使用 [[01-我的/03-常工院/00-课程体系/cs-游戏引擎(UECS)/04-UE-知识体系/纹理图集/纹理图集|UE/视觉/纹理/技术/纹理图集(Atlas)]] 优化小物件，通过性能工具诊断并解决 Overdraw 与显存溢出。
- **达成成果**：在保持视觉品质的前提下，达成目标平台的帧率要求（如 60FPS）。

---

> [!TIP] 实践指南
>
> **" 先视觉，后性能 " 的螺旋上升法**：
>
> 在 UE 实践中，建议先用 **HDRI + Nanite** 建立最高的视觉标杆。随后，通过引入 **Atlas** 和 **Mipmap** 的概念进行 " 瘦身 "。这种从 " 极致视觉 " 到 " 极致性能 " 的逆向推导，能让你更深刻地理解为什么某一张贴图必须是 2048 x 2048，以及为什么材质必须要打包 [[RMA贴图|RMA]]。

---

## 4. 关键成果评估 Checklist

- [ ] **规范度**：所有贴图符合 2 的幂次方且完成了 RMA 打包。
- [ ] **流畅度**：在大规模部署 Nanite 资产后，Draw Calls 处于健康区间。
- [ ] **准确度**：HDRI 投影中心与地面资产比例完美匹配，阴影方向一致。
- [ ] **稳定性**：通过 `stat streaming` 验证，无红色显存警告。
