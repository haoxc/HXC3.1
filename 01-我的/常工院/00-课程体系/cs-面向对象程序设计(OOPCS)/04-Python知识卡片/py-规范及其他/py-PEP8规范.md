在 Python 中，命名规范主要遵循**PEP 8 风格指南**，不同类型的对象（变量、函数、类、模块等）有不同的命名规则，目的是提高代码的可读性和一致性。以下是核心命名规范的整理：
### 一、变量与函数命名（蛇形命名法 `snake_case`）
- **规则**：全部小写，单词之间用下划线 `_` 分隔。
- **示例**：
    - 变量：`user_name`、`total_count`、`is_valid`
    - 函数：`calculate_average()`、`read_file()`、`get_user_info()`
### 二、类命名（大驼峰命名法 `PascalCase`）
- **规则**：每个单词首字母大写，无下划线。
- **示例**：
    - 普通类：`UserProfile`、`DataProcessor`、`HttpRequest`
    - 异常类：`ValueError`（继承自`Exception`，通常以`Error`结尾）
### 三、模块与包命名
- **模块名**：全部小写，可含下划线（简洁优先）。
    - 示例：`data_utils.py`、`logger.py`
- **包名**：全部小写，通常不含下划线（保持简洁）。
    - 示例：`requests`（第三方库）、`my_project`（自定义包）

### 四、常量命名（全大写蛇形命名法）
- **规则**：全部大写，单词之间用下划线 `_` 分隔。
- **示例**：
    - `MAX_CONNECTIONS`、`PI = 3.14159`、`DEFAULT_TIMEOUT`

### 五、特殊命名（用于标识用途或范围）

| 命名格式      | 含义                                                       | 示例                 |     |
| --------- | -------------------------------------------------------- | ------------------ | --- |
| `_xxx`    | 单下划线开头：**内部使用**（提示开发者 “不建议外部访问”，无强制限制）                   | `_internal_func()` |     |
| `__xxx`   | 双下划线开头：**名称修饰**（避免子类覆盖，Python 会自动改写名称）                   | `__private_attr`   |     |
| `__xxx__` | 前后双下划线：**特殊方法 / 变量**（Python 内置的 “魔法方法”，如类的初始化`__init__`） | `__name__`（模块名）    |     |
### 六、其他注意事项
1. **避免关键字**：不要使用 Python 的保留关键字（如`if`、`for`、`class`等）作为命名。
    - 错误示例：`class = "A"`（`class`是关键字）
2. **见名知意**：命名要清晰表达含义，避免模糊的缩写。
    - 推荐：`student_age`
    - 不推荐：`sa`（缩写难以理解）
3. **类型提示命名**：类型别名（Type Alias）通常用大驼峰命名，与类命名一致。
    - 示例：`UserId = int`、`UserDict = dict[str, str]`
### 总结
Python 命名规范的核心是 **“清晰优先，风格统一”**。遵循这些规则能让代码更易读、易维护，尤其在团队协作或开源项目中，一致性的命名是代码质量的重要保障。
