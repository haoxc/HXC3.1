## 概述

在 Python 中，**不可变类型（Immutable Types）** 指的是一旦创建，其内部数据（值或结构）就无法被修改的类型。
当你尝试 “修改” 这类对象时，实际上会创建一个新的对象，而原对象的内容保持不变。
### 常见的不可变类型及特点：
1. **数值类型**：整数（`int`）、浮点数（`float`）、复数（`complex`）
    - 示例：`x = 10`，若执行 `x = 20`，并非修改原`10`的值，而是让变量`x`指向新的整数`20`，原`10`仍存在（直到被垃圾回收）。
2. **字符串（`str`）**
    - 字符串的每个字符不可单独修改，任何 “修改” 操作（如拼接、替换）都会生成新字符串。
    - 示例：`s = "abc"`，`s[0] = "A"` 会报错；`s = s.replace("a", "A")` 会创建新字符串 `"Abc"`。
3. **元组（`tuple`）**
    - 元组的元素数量和引用不可修改（不能增删元素，也不能替换元素）。
    - 注意：若元组包含**可变类型元素**（如列表），该元素的内部内容可修改（但元组本身的结构不变）。
    - 示例：`t = (1, [2, 3])`，`t[0] = 4` 报错，但 `t[1].append(4)` 合法（列表`[2,3]`被修改为`[2,3,4]`，元组结构仍为`(1, [2,3,4])`）。
4. **布尔值（`bool`）**
    - 仅有`True`和`False`两个固定值，不可修改。
5. **冻结集合（`frozenset`）**
    - 集合的不可变版本，创建后不能增删元素，支持集合运算（如交集、并集），但自身不可变。
    - 示例：`fs = frozenset({1, 2, 3})`，`fs.add(4)` 会报错。
### 不可变类型的核心特性：
- **可哈希（Hashable）**：不可变类型的对象有固定的哈希值（`hash()` 可计算），因此可作为[[py-字典(Dict)|字典(Dict)]] 的键（`key`）或[[py-集合(Set)|集合(Set)]] 的元素。
- **线程安全**：由于不可修改，多线程环境下无需担心并发修改导致的问题。
- **内存优化**：相同值的不可变对象可能共享内存（如小整数池、字符串驻留机制）。
### 与可变类型的关键区别：
- 可变类型（如列表`list`、字典`dict`、集合`set`）可直接修改内部数据，操作的是原对象；
- 不可变类型 “修改” 时会生成新对象，原对象不变。
