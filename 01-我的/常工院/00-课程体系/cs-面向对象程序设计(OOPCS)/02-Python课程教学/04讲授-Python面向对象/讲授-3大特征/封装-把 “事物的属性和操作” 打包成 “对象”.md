## 概述
**核心逻辑**：每个 “对象” 就像一个 “独立的素材包”，里面既有 “属性”（素材的特征），也有 “方法”（对素材的操作），对外只暴露必要的接口，隐藏内部细节。

**数媒场景示例**：
- 比如 PS 中的 “图层” 就是一个典型的 “对象”：
    - **属性**：尺寸（宽 × 高）、透明度、混合模式（如 “正片叠底”）；
    - **方法**：移动（改变位置）、缩放（调整大小）、合并（与其他图层融合）；
    - **封装性**：你不需要知道 PS 底层如何计算 “正片叠底” 的像素混合，只需点击 “混合模式” 选择即可（隐藏细节，暴露接口）。
    - 
- 在代码中，“**图片对象**” 可以这样封装：
    ```python
    class Image:  # 定义“图片”类（对象的模板）
        # 属性：图片的特征
        def __init__(self, width, height, format):
            self.width = width  # 宽度
            self.height = height  # 高度
            self.format = format  # 格式（PNG/JPG）
        
        # 方法：对图片的操作
        def resize(self, new_width, new_height):
            self.width = new_width
            self.height = new_height
            print(f"图片已调整为{new_width}×{new_height}")
        
        def convert_format(self, new_format):
            self.format = new_format
            print(f"图片已转换为{new_format}格式")
    
    
    ```

> 💡**封装**是将对象的状态（`属性`）和 `行为`（方法）包裹在一起的过程。

## 价值
- **隐藏实现细节**：封装的主要目的是隐藏对象的状态和实现。这意味着外部对象不能直接访问对象内部的状态或实现。这样可以防止外部对象意外地修改对象状态。
- **提供公开的接口**：虽然对象的内部状态是私有的，但可以通过公开的方法或函数提供外部访问的接口。这些方法定义了对象的行为。
- **数据保护**：通过确保只有经过验证和控制的方式才能访问或修改数据，封装提供了一种数据保护机制。
- **易于修改和维护**：由于实现细节被隐藏，所以更容易修改内部实现而不影响使用该对象的外部代码。

## 示例
**示例:** 
```python
class Person:
    def __init__(self, name, age):
        self.name = name  # 公开属性（可直接访问）
        self.age = age  # 私有属性（用双下划线标记，外部不能直接访问）
    
    # 对外暴露的接口（方法），用于安全地获取/修改私有属性
    def get_age(self):
        return self.__age
    
    def set_age(self, new_age):
        if new_age > 0 and new_age < 150:  # 增加校验逻辑
            self.__age = new_age
        else:
            print("年龄输入无效")

# 使用
p = Person("小明", 20)
print(p.name)  # 允许直接访问公开属性 → 小明
p.__age =20
print(p.get_age())  # 通过接口访问私有属性 → 20
p.set_age(200)  # 触发校验，拒绝修改 → 年龄输入无效
```

- 属性
	- `公开属性`: 外部能够访问
	- `私有属性`: （用*双下划线*标记，*外部不能直接访问*）
	- `保护属性`: （用*单下划线*标记，*外部不能直接访问*, 但继承者可以访问）
