## 概要
- 这项特性旨在简化程序的*入口点*编写，尤其是对于那些用于学习、小型项目、或是简单工具的代码。
- 顶级语句消除了传统 `Main` 方法周围的“样板代码”，让你能够更快速、更直观地开始编写代码。

```csharp
using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}
```

这个代码片段包含了`类`和`方法`定义，这对于新手来说可能是一种认知负担。而有了顶级语句，相同的程序可以写成:
```csharp
using System;
Console.WriteLine("Hello, World!");
```

>[!important] 要点
> - **省略 Main 方法**：你不需要显式定义一个 `Main` 方法。它在背后被隐式创建。
> - **命名空间和类级别的元素**：你仍然可以在使用顶级语句的文件中定义命名空间、类、方法等，但它们必须位于顶级语句之后。
> - **参数访问**：在顶级语句中，如果你需要访问传递给应用程序的命令行参数，你可以使用 `args` 变量，就像在传统的 `Main` 方法中一样。编译器在幕后处理这个变量，无需事先声明即可直接使用。
