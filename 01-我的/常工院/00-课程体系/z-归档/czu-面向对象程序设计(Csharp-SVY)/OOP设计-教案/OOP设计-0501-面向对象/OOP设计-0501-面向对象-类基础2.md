## 属性
### 定义属性
- 属性（Properties）是类的成员之一，用于提供对类的字段的安全访问。
- 属性可以让你控制字段的`读取`（get）和`写入`（set）操作，从而为数据的访问提供更高层次的封装和保护。

> [!important] 特性
> - **封装**：属性提供了一种机制来获取和设置私有字段的值，而不必直接暴露这些字段。
> - **验证**：在设置属性值时，可以添加逻辑来验证数据。
> - **简化语法**：属性使用与字段相似的语法，但同时提供了方法的灵活性。


```csharp
public class Student{
	//字段(成员)
	private   string name;  //姓名
	private   int    age;   //年龄

}

```

如何改造
```csharp
public class Student{
	private string name;
 
	//自动实现的属性
	public   int     {get; set;}   //年龄
	public   string Name{
		get{return  this.name;} 
		set{
			this.name  = value;
			if (string.IsNullOrEmpty(value)) throw new ArgumentException("Name cannot be null or empty"); 
			name = value;
		}
	}  //姓名
	//...
}
```
- 在上面的例子中，`Age` 是一个自动实现的属性，它自动创建一个私有的、匿名的`后备字段`。而 `Name` 属性则包含逻辑来确保名字不为空。
- **value** 是set方法的`参数` 
### 只读和只写属性

- **只读属性**：只有 `get` 访问器。
- **只写属性**：只有 `set` 访问器。

### 使用属性
```csharp

 void ExerciseAttribute()
{
	Person person = new Person();

	person.Name = "Alice"; // 设置属性
	person.Age = 30;       // 设置属性

	Console.WriteLine($"Name: {person.Name}, Age: {person.Age}");
}


```


## 静态函数
- 静态函数（也称为静态方法）是属于类本身而不是*类的实例*的方法。
- 这意味着您*无需创建类的实例即可调用静态函数*。
- 它们通常用于执行不依赖于对象状态的操作。


### 静态函数的特点：
> [!important] 要点
> - **属于类**：静态函数属于类，而不是类的`特定实例`。
> - **无法访问非静态成员**：静态函数只能访问类的静态成员，无法访问非静态成员。
>- **全局访问**：可以通过类名直接调用静态函数，而无需创建对象实例。
>- **应用场景**：常用于工具函数、辅助方法，或当方法的执行不依赖于对象的状态时。

**示例:**
```csharp
public class Utility
{
    // 静态方法
    public static int Add(int a, int b)
    {
        return a + b;
    }

    // 另一个静态方法
    public static int Multiply(int a, int b)
    {
        return a * b;
    }
}
```

**调用:**
```csharp

static void TestUtility()
{
	int sum = Utility.Add(5, 3);        // 调用静态方法
	int product = Utility.Multiply(4, 2); // 调用静态方法

	Console.WriteLine($"Sum: {sum}, Product: {product}");
}


```
