## gstack 核心观点
1. **项目定位**：Y Combinator 总裁 Garry Tan 开源的 **Claude Code 专用 AI 编程工作流**，上线6天获2万+ Star，是一套把通用 AI 变成专业开发团队的角色化工作流系统。
2. **核心理念**：**拒绝让单一 AI 包办一切**，避免产品、技术、设计、代码、发布混为一谈导致结果平庸；通过**角色分工+明确档位**，让 AI 各司其职、专精一项任务。
3. **核心机制**：提供**12个Skill**，对应CEO、技术负责人、设计师、资深工程师、发布工程师等角色，覆盖**产品规划→技术方案→编码→代码审查→测试→发布→复盘**全开发生命周期。
4. **技术优化**：浏览器自动化不采用 MCP，而是**编译为 CLI 二进制**，通过命令行参数+纯文本输出对接 Claude Code 的 Bash 能力，**大幅降低 Token 开销**。
5. **使用范式**：按流程调用 Skill 形成闭环：需求描述 → CEO 评审产品方向 → 技术负责人定方案 → 编码 → 偏执级代码审查 → 一键发版 → 系统化测试 → 周复盘，**越用越智能**。
6. **价值**：把个人开发者升级为**指挥 AI 团队的管理者**，不用全能，只需知道**何时调用哪个专业角色**，显著提升复杂项目的完整性、质量与上线效率。

# gstacks.org 官网
核心是介绍由 Y Combinator 总裁 Garry Tan 打造、面向 **Claude Code** 的 AI 编程工作流工具，将单一 AI 助手转化为**虚拟软件开发团队**，覆盖从产品规划到上线复盘全流程。

## 一、项目基础信息
- 作者：Garry Tan（YC 总裁兼 CEO）
- 协议：MIT 开源许可，当前版本 v0.4.1
- 热度：GitHub 获 **16,000+ Star、1,800+ Fork**，开源后快速爆红
- 定位：不是普通提示词包，而是**角色化、全流程、可落地**的 AI 开发工作流系统

## 二、核心理念：解决 AI 编码平庸化问题
传统 AI 助手用**单一通用模式**包办所有任务，导致输出缺乏深度、无产品思考、评审质量不稳定、无视觉验证、发布与 QA 高度手动。
GStack 理念：**AI 需明确认知档位**，规划≠评审≠发布≠严谨工程，模糊边界只会产出平庸结果，要让 AI 按需切换专业角色。

## 三、核心能力：9 大 Skill 覆盖全开发周期
通过斜杠命令激活**9 个专家角色**，形成闭环工作流：
1.  /plan-ceo-review：创始人/CEO，从用户视角验证产品方向
2.  /plan-eng-review：技术负责人，输出架构与流程图，敲定技术方案
3.  /review：偏执资深工程师，做生产级代码审计（N+1、竞态、安全边界等）
4.  /ship：发布工程师，一键同步、测试、推分支、开 PR
5.  /browse：QA 工程师，浏览器自动化可视化校验
6.  /qa：QA 负责人+修复工程师，diff 感知、自动测-修-验
7.  /qa-only：纯 QA 报告，不修改代码
8.  /setup-browser-cookies：会话管理，免登录测试鉴权页面
9.  /retro：工程经理，迭代复盘、产出与效率分析

## 四、技术架构（关键亮点）
- 浏览器自动化：**持久化 Chromium 守护进程**，首启约 3s，后续指令 100–200ms，比 MCP 快约 20 倍，Token 开销极低
- 部署：Bun 编译为 **58MB 单二进制文件**，无依赖、本地运行、不污染 PATH
- 安全：localhost 绑定、令牌鉴权、Cookie 加密、防注入、只读数据库
- 并发：支持 **10 个隔离会话并行**，一人可同时驱动多任务流水线

## 五、标准工作流（开箱即用）
需求描述 → /plan-ceo-review（产品方向压力测）→ /plan-eng-review（技术方案）→ 编码 → /review（深度审查）→ /ship（一键发版）→ /qa（全量验证）→ /retro（周复盘）

## 六、集成与效果
- 集成 Greptile 自动代码评审，双层校验、减少漏检与误报
- 作者实测：**日均万行代码、周均百个 PR**，兼顾产量与生产质量
- 与 Copilot 等互补：前者专注代码生成，GStack 覆盖**全生命周期+可视化 QA**

## 七、安装与使用
- 依赖：Claude Code、Git、[[Bun]] v1.0+
- 支持：macOS / Linux（x64/arm64）
- 两步安装：克隆仓库→执行 setup；团队可嵌入仓库共享使用
- 升级：内置 /gstack-upgrade 命令，支持自动更新

## 八、目标用户
重度 Claude Code 用户、追求**工程规范与交付效率**的开发者/小团队，把 AI 从“代码工具”升级为“虚拟开发团队”。
