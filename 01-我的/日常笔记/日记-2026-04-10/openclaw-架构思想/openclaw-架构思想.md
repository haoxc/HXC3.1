# OpenClaw 资源管理架构解析

图片展示了 **OpenClaw** 平台的核心资源管理菜单，它勾勒出了一个现代 **智能体平台(Agentic Platform)** 的标准功能分层：

## 1. 核心资源模块 (Core Modules)

- **智能体(Agents)**：系统的核心执行主体。代表了具备特定身份、目标和能力的独立 AI 实体。
    
- **[[指令(Instructions)]]**：即系统提示词(System Prompts)。定义了智能体的角色定位、行为准则以及任务处理逻辑。
    
- **技能(Skills)**：智能体的原子能力集。通常是封装好的特定功能（如：网页抓取、文本格式化），供智能体在任务中调用。
    

## 2. 扩展与集成 (Extensions & Integration)

- **插件(Plugins)**：外部工具的集成接口。这通常是 **MCP(Model Context Protocol)** 服务器或第三方 API 的接入点，让智能体*具备操作物理世界的能力*（如 Chrome 控制、文件读写）。
- **钩子(Hooks)**：事件驱动逻辑。用于监听特定状态或行为（如任务开始、完成或报错），并触发预定义的自动化动作，实现系统的灵活性。
    

## 3. 编排与协同 (Orchestration)

- **[[智能体工作流(Agentic Workflows)]]**：高阶任务编排。将多个智能体、技能和插件按照逻辑顺序（循环、分支、并行）串联起来，解决单体智能体无法完成的复杂长路径任务。
