# 鼠标模拟技术全景指南

鼠标模拟是桌面自动化的执行核心，其本质是**伪造输入信号**。根据模拟发生的层级，技术路径可分为软件层、驱动层和硬件层。

---

## 1. 技术路径分类

| 维度      | 实现方式         | 典型工具/API                     | 优点          | 缺点            |
| ------- | ------------ | ---------------------------- | ----------- | ------------- |
| **软件级** | 调用 OS 标准 API | `SendInput`, `mouse_event`   | 兼容性好，开发快    | 易被检测，受应用权限限制  |
| **驱动级** | 编写或调用过滤驱动    | DD 驱动, WinIo, 逻辑模拟           | 隐蔽性高，绕过应用检测 | 签名门槛高，系统稳定性风险 |
| **硬件级** | 物理设备模拟 HID   | Arduino, KMBox, Raspberry Pi | 绝对物理级，不可检测  | 需额外硬件，成本及部署复杂 |

> **核心凝练**：根据系统交互深度，将模拟技术划分为逻辑欺骗（软件）、系统伪装（驱动）和物理仿真（硬件）三个维度。

---

## 2. 核心实现原理

### A. 操作系统消息队列

大多数自动化工具（如 [[PyAutoGUI]]）通过调用操作系统的 API 将事件插入输入队列。这些事件带有“LLMHF_INJECTED”标志位，容易被反作弊或安全软件识别。

### B. 驱动截获与注入

通过在 HID（人机交互设备）堆栈中插入过滤驱动，模拟真实的底层硬件中断。这种方式对于目标软件而言，接收到的信号与真实鼠标物理移动几乎无异。

### C. 视觉反馈环 (AI 驱动)

在 LAA (Large Action Model) 背景下，鼠标模拟不再依赖固定坐标，而是通过：

1. **视觉感知**：截图识别目标。
2. **轨迹规划**：生成模拟人类操作的非线性路径。
3. **动作执行**：调用底层接口完成点击。

> **核心凝练**：现代鼠标模拟已从单一的信号注入演变为“视觉感知-路径规划-底层执行”的闭环反馈系统。

---

## 3. 实践指南

> [!IMPORTANT] **技术选型策略**
> 1. **办公自动化**：优先选择 **Python (PyAutoGUI/Clicknium)** 或 **RPA 工具**。此时效率和开发成本是核心，无需担心检测。
> 2. **高强度对抗环境（如游戏/安全系统）**：必须使用 **硬件级 (KMBox/Arduino)**。物理层面的隔离是规避封禁的唯一底层逻辑。
> 3. **稳定性要求**：如果涉及多账户、多环境部署，尽量避免依赖坐标的模拟，应采用 **基于 UI 元素识别** 的模拟方式，以应对窗口缩放和分辨率差异。
> 
> 

> **核心凝练**：选型应遵循“最小必要原则”，在满足环境对抗强度的前提下，优先选择维护成本最低的软件级方案。

---

## 4. 模拟的“真实感”优化

为了让模拟更接近真人，需在代码中引入：

* **贝塞尔曲线 (Bézier Curve)**：模拟非直线的鼠标移动轨迹。
* **随机延迟 (Jitter)**：在点击动作前后加入 20ms-100ms 的正态分布随机等待。
* **分段加速**：模拟人类“启动-加速-减速-微调”的物理运动规律。

> **核心凝练**：通过引入数学噪声与物理运动模型，将离散的逻辑指令转化为具有生物特征的连续信号。
