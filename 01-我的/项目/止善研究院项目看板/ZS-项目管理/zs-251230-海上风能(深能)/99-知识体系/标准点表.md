## 术语
在数字孪生项目实施中，“标准点表”是==连接物理实体与虚拟模型的**数据契约**==。截至2026年，随着《[[信息技术车间数字孪生参考架构]]》（GB/T 45873-2025）等国家标准的正式实施，点表的编制已趋向标准化

一份规范的数字孪生标准点表通常包含以下核心维度：

### 1. 基础识别信息

用于唯一标识传感器或设备点位，确保虚实对应准确：
- **点位编码 (ID)：** 全局唯一标识符（如基于测点、设备资产编号）。
- **测点名称：** 易读的中文描述，如“1号泵出口压力”。
- **物理对象：** 关联的物理实体名称（如：空压机_01）。
- **所属系统：** 如暖通系统、动力系统、生产控制系统等。

### 2. 模型绑定信息 (Twin Mapping)
这是数字孪生点表区别于传统SCADA点表的关键：
- **三维模型ID：** 对应BIM/模型中的构件唯一代码。
- **空间坐标：** 该点位在孪生场景中的 X, Y, Z 位置坐标。
- **交互属性：** 该点是仅用于**展示**（只读）还是可**反向控制**（读写）。 
### 3. 数据属性规范
定义实时数据的传输与解析格式：
- **数据类型：** 整型 (Int)、浮点型 (Float)、布尔型 (Boolean) 或枚举。
- **物理单位：** 标准化单位（如 ℃、MPa、kW·h）。
- **量程范围：** 定义数据的有效阈值，用于异常诊断和告警预测。
- **采集频率：** 如 500ms、1s 或变化上送。

### 4. 通信与接口参数
- **协议类型：** 如 MQTT、OPC UA、Modbus TCP 或 HTTP RESTful。
- **点位路径/地址：** 原始协议中的寻址地址。
- **更新触发机制：** 周期性上报、阈值触发或心跳包。 

### 5. 常见业务场景示例

| 类别       | 典型点位 (Points)   | 孪生用途          |
| -------- | --------------- | ------------- |
| **智慧工业** | 转速、温度、电流、振动频率   | 预测性维护、故障模拟    |
| **智慧城市** | 客流数、空气质量、路口拥堵指数 | 城市运行态势感知、仿真调优 |
| **智慧水利** | 水位、PH值、溶解氧、流量   | 洪涝预警、水质回溯分析   |

### 建议与标准参考
- **执行标准：** 建议参考 **GB/T 45873-2025**（车间数字孪生）及各行业团体标准（如水利、电力行业技术规范）。
- **管理建议：** 2026年的趋势是利用“数据底座”实现点位的自动发现与语义化建模，不再单纯依赖手动填表，而是通过元数据管理系统进行同步
