# 总结

Arcadia是一种面向系统和架构工程的模型驱动工程方法，由Capella建模工具支持，旨在帮助工程师理解客户真实需求、定义和共享产品架构、早期验证设计并支持集成、验证、验证和合格（IVVQ）过程。

# 摘要

Arcadia是由[[01-我的/01-项目管理/止善研究院项目看板/ZS-解决方案/zs方案-数字化三维智能管理(260116)/02-方案调研/数字化方案/泰雷兹(Thales)/泰雷兹(Thales)]]开发的一种模型驱动系统工程方法，专注于系统、硬件和软件架构设计。它通过分析运营需求、明确系统需求、开发逻辑架构和物理架构，以及建立开发和IVVQ的合同，来指导工程活动。Arcadia采用面向视角的方法，强调需求工程、运营和功能需求分析、架构构建和验证之间的平衡。该方法支持企业范围的协作和共同工程，适用于不同的工程水平和领域，并且可以适应多种生命周期模型，如迭代、增量、自上而下、自下而上等。Arcadia方法的核心在于通过模型转换和链接，以及通过Eclipse Capella™工作台建立的模型，来管理复杂性和规模，同时支持与客户和子系统工程的协作。此外，Arcadia还提供了一系列在线文档和参考书籍，以帮助用户深入了解其原则和概念。

# 观点
- **Arcadia是一种模型驱动的工程方法**，专注于系统和架构设计，支持系统、硬件和软件的协同工程。
- 该方法强调**运营需求分析**，以确保系统定义与其真实的运营使用和[[Integration, Validation, Verification, Qualification (IVVQ)|IVVQ]]条件相符。
- **系统需求的正式化**涉及定义系统如何满足运营需求，包括功能、服务、交换、非功能性约束等。
- **逻辑架构的定义**涉及构建系统的粗粒组件分解，以做出重要的工程决策，并考虑所有主要的非功能性约束。
- **物理架构的定义**进一步细化逻辑架构，引入资源组件和技术服务，以准备开发和IVVQ。
- **建立策略**涉及为开发和IVVQ定义组件集成合同，确保所有组件都符合预期的属性。
- Arcadia支持**协同工程、分包和多层次工程**，允许在不同的工程水平应用相同的方法。
- 方法可以根据不同的领域和产品进行**定制和适应**，包括定义参考架构、视角、工程规则等。
- Arcadia适用于多种**生命周期和工作分享方案**，不仅限于自上而下的方法，还包括迭代、增量、自下而上等。
- 提供了一系列**在线文档和参考书籍**，帮助用户深入了解Arcadia的原则和概念。
