### 一句话总结

将场景要素标准化为“属性（Context）- 状态（Connection）- 行为（Control）”三位一体的对象模型，为 AI 自动化构建与交互提供结构化支撑。

---

## 一、 核心对象模型定义

我们将场景内的所有可视元素抽象为具有继承关系的“对象类”，确保所有对象具备统一的空间基准，同时针对不同类型扩展特有的物联与交互能力。

### 1. 基础场景对象 (Base Scene Object)

_所有空间实体的基类，提供最基本的空间描述。_

- **属性 (Properties)**
    - **标识信息：** `UUID`, `Name`, `Tag` (用于 AI 语义分组)。
    - **空间变换 (Transform)：**
        - `Position` (X, Y, Z)
        - `Rotation` (Pitch, Yaw, Roll)
        - `Scale` (X, Y, Z)
- **操作 (Operations)**
    - `SetVisible(bool)`: 显示/隐藏。
    - `TransformTo(pos, rot, duration)`: 平滑移动或旋转至目标状态。

---

### 2. 物联设备 (IoT Device)

_继承自基础对象，平台的核心实体，承载业务数据与控制逻辑。_

- **属性 (Properties)**
    - **静态档案 (Context)：**
        - `DeviceType`: 枚举值 [HVAC, Lighting, Security, Sensor...]
        - `VendorInfo`: 供应商 ID, 设备型号。
        - `AssetID`: 资产管理编码。
    - **视觉表现 (Visual)：**
        - `MaterialMode`: [Default, X-Ray, Heatmap, Alarm]
        - `ColorOverlay`: 叠加色（如：报警红，离线灰）。
        - `Opacity`: 透明度 (0.0 - 1.0)。
    - **动态能力 (Animation)：**
        - `AnimList`: 支持的动作列表 (e.g., ["Open", "Close", "Rotate"]).
        - `CurrentState`: 当前动画帧位置/状态。
- **操作 (Operations)**
    - **状态控制：**
        - `SetVisualMode(mode, color)`: 切换视觉渲染模式（如：进入透视模式查看内部结构）。
    - **动作执行：**
        - `PlayAnim(clipName, speed, loop)`: 驱动设备动作（如：以 2x 速度播放风扇转动）。
        - `SyncState(telemetryData)`: **核心逻辑**，将 IoT 实时数据映射为设备状态（如：温度 > 50°C -> 变红）。

---

### 3. 智能摄像机 (Smart Camera)

_场景的“观察者”，决定用户的交互视角与漫游体验。_

- **属性 (Properties)**
    - **镜头参数：**
        - `FOV`: 视场角 (60° - 120°)。
        - `Projection`: 投影模式 [Perspective(透视), Orthographic(正交)]。
        - `ClippingPlane`: 裁剪范围 (Near, Far)。
    - **交互参数：**
        - `OrbitCenter`: 轨道旋转中心点。
        - `Min/MaxDistance`: 缩放限制范围。
- **操作 (Operations)**
    - **视角变换：**
        - `FlyTo(targetID, offset, duration)`: 智能聚焦到某个设备（自动计算最佳观察距离）。
        - `SwitchMode(mode)`: 切换 2D/3D 视图模式。
    - **巡航控制：**
        - `SetPath(pathID, speed)`: 绑定巡检路径。
        - `LookAt(coordinate)`: 锁定注视点。

---

### 4. 辅助交互对象 (Auxiliary Interaction Objects)

_增强场景信息密度与导航能力的非物理实体。_

#### A. 信息面板 (Information Panel)

- **类型：** `WebOverlay` (2D DOM), `WorldSpaceUI` (3D 空间跟随)。
- **操作：**
    - `InjectData(json)`: 注入实时数据。
    - `Toggle(bool)`: 开关状态。
    - `AnchorTo(targetID)`: 挂载到特定设备上方。

#### B. 空间路径 (Spatial Path)

- **类型：** `Patrol` (巡检线), `Evacuation` (逃生路线), `Logistics` (物流轨迹)。
- **操作：**
    - `VisualizeFlow(speed, color)`: 开启粒子流动效果（模拟人流或气流）。
    - `SetWaypoints(points[])`: 动态更新路径点。

## 二、 对象定义结构示例 (JSON-Schema)

为了便于开发落地，建议使用如下 JSON 结构定义一个标准的物联设备：

```
{
  "entity_id": "pump_01_MainHall",
  "class": "IoTDevice",
  "context": {
    "transform": { "pos": [100, 0, 50], "rot": [0, 90, 0], "scale": [1, 1, 1] },
    "identity": { "type": "Pump", "model": "Grundfos-X9" }
  },
  "visual": {
    "mesh_url": "assets/pump_v2.glb",
    "default_material": "mat_metal_brushed"
  },
  "capabilities": {
    "animations": ["run", "stop", "fault"],
    "telemetry_map": {
      "rpm": "anim_speed_multiplier",  // 映射逻辑：转速控制动画速度
      "status": "color_state_machine" // 映射逻辑：状态控制颜色
    }
  }
}
```

## 实践指南

1. **原子化解耦：** 即使是复杂的组合设备（如一条生产线），也应拆解为独立的“原子设备”进行定义，通过 `ParentID` 组合。这样 AI 在进行故障诊断时，能精确指向具体的“电机”而不是模糊的“生产线”。
2. **摄像机阻尼：** 在设计 `FlyTo` 或 `聚焦` 操作时，务必加入“摄像机阻尼 (Damping)”参数。不要瞬间切镜，而是使用 Ease-in-out 插值曲线，避免用户产生 3D 眩晕感。
3. **LOD 分级策略：** 对于“设备”对象，建议定义 LOD0 (高模) 和 LOD1 (包围盒/图标)。当摄像机距离较远时，自动切换为 LOD1 或仅显示“信息面板”，降低渲染开销并减少视觉噪点。
